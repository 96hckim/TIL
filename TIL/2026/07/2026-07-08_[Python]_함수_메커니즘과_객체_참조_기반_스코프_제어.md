### 1. 함수의 정의, 호출 및 리턴 메커니즘

- return의 이중적 역할 :
- 데드 코드 (Dead Code) : return문 바로 아랫줄에 작성된 코드들은 실행 흐름이 절대로 도달할 수 없으므로 무시됨 (예: func4-2 출력문은 실행되지 않음).
- 무반환 함수의 실체 : 파이썬의 함수는 내부에 명시적인 return이 없거나 명사 없이 return 키워드만 단독으로 쓰인 경우, 함수가 끝날 때 자동으로 None 객체를 반환하도록 설계되어 있음.
```python
def func1():
    print("실행")
    return       # 함수 즉시 종료 (None 반환)
    print("미실행") # 데드 코드 (무시됨)

print(func1())   # 출력: 실행 -> None (리턴값이 없으므로 None 출력)
```

### 2. 동적 인자(Parameter) 처리와 복수 리턴의 비밀

- 파이썬의 동적 타이핑 (Dynamic Typing) : 파이썬 함수는 매개변수의 자료형을 미리 고정하지 않음. 따라서 func2(a, b)에 숫자, 문자열, 리스트를 넣어도 내부 연산자(+)가 지원하는 연산이라면 에러 없이 유연하게 처리함. 단, 자료형이 다른 녀석끼리("kim" + 10) 묶으면 TypeError가 발생함.
- 복수 값 리턴의 실체는 '패킹' : 파이썬 함수는 return a+b, a-b 처럼 콤마(,)를 이용해 여러 개의 값을 동시에 리턴할 수 있음. 하지만 내부적으로는 여러 값이 하나의 Tuple 객체로 자동 패킹되어 단 하나의 컨테이너로 리턴되는 메커니즘임. 당연히 호출한 좌변에서 변수 개수를 맞춰 언패킹 수령이 가능함.
```python
def func(a, b):
    return a + b, a - b # 하나의 튜플로 묶여서 리턴됨

result = func(10, 3)
print(result) # 출력: (13, 7) -> 튜플 객체 하나임
```

### 3. [★초핵심] 변수의 스코프(Scope)와 global 대참사 예방

- 함수 내부의 할당(Assignment) 법칙 : 함수 내부에서 변수에 값을 대입(cnt += 1 또는 cnt = 1)하는 행위가 포착되면, 파이썬 인터프리터는 그 변수를 바깥의 전역 변수가 아니라 함수 내부에서만 살아 숨 쉬는 '지역 변수(Local Variable)'로 자동 임명함.
- UnboundLocalError의 원인 ([6-7]) : 바깥에 cnt = 0이 있어도, 함수 내부에서 cnt += 1을 실행하면 파이썬은 내부의 지역 변수 cnt를 먼저 찾음. 하지만 아직 선언도 안 된 내부 cnt에 +1을 시도하는 꼴이 되므로 에러를 뿜으며 주저앉음.
- global 키워드의 역할 ([6-8]) : 함수 내부에서 바깥 영역의 전역 변수를 직접 수정하거나 완전히 새로 선언하여 바깥세상에 박제하고 싶을 때 명시적으로 사용하는 치트키.
```python
# [6-7] ⚠️ 대참사 에러 코드
cnt = 0
def func():
    cnt += 1 # UnboundLocalError 발생! (지역 변수 cnt가 할당 전에 참조됨)

# [6-8] ✨ global을 통한 안전한 전역 참조 및 제어
def f2():
    global a, b # a와 b는 이제 바깥 세상의 전역 변수와 완전히 동기화됨
    a = 10
    b = 20      # 바깥에 b가 없었더라도 전역 공간에 b가 새로 박제됨
    c = 30 + d  # 할당 연산(=)이 없는 d는 global 선언 없이도 바깥의 d(5)를 알아서 잘 읽어옴(Read-only)

a, c, d = 1, 3, 5
f2()
print(a, b, c)  # 출력: 10 20 3 (f2 내부의 c는 지역 변수라 파괴되었으므로 바깥의 c인 3 유지)
```

### 💡 부족했던 디테일 한 스푼 보정 (추가/수정 정보)

1. 읽기(Read)와 쓰기(Write)의 스코프 차이 : [6-8]의 c = 30 + d 예제에서 보듯, 함수 내부에서 바깥 전역 변수의 값을 단순히 읽어오기만 할 때(d)는 global 키워드가 전혀 필요 없습니다. 오직 값을 수정하거나 새로 대입(write)할 때만 global 선언이 강제됩니다.
1. global c 누락의 결과 이해 : 함수 내부에서 c = 30 + d 연산으로 지역 변수 c가 새로 만들어졌기 때문에, 함수가 끝나는 순간 내부의 c는 메모리에서 증발합니다. 따라서 맨 아랫줄 print(a, b, c)에서 호출하는 c는 함수 실행 전 선언해 둔 최초의 전역 변수 3이 그대로 출력되는 스코프 분리 메커니즘을 명확히 인지하셔야 합니다.
### 4. 파이썬의 인자 전달 기법: Call by Object Reference

- 객체 참조 전달 : 파이썬은 인자를 넘길 때 값이 복사되거나 주소 자체가 넘어가는 게 아니라, 객체의 메모리 주소(ID) 포인터를 그대로 복사해서 넘겨줍니다. 이 때문에 넘겨주는 데이터가 불변이냐 가변이냐에 따라 동작이 완전히 갈립니다.
- 불변 객체(Immutable) 전달 ([6-9-1]) : 정수, 문자열, 튜플을 함수에 넘기면 함수 내부에서 값을 바꾸는 순간(a = 100), 완전히 새로운 객체가 생성되면서 지역 변수의 링크가 끊어집니다. 따라서 바깥 세상의 원본 변수에는 아무런 영향이 없습니다.
- 가변 객체(Mutable) 전달 ([6-9-2]) : 리스트, 세트, 딕셔너리를 함수에 넘기고 내부 알맹이를 수정(x[0] = 100, .append())하면, 같은 주소록 장부를 공유하고 있었기 때문에 바깥에 있는 원본 객체가 실시간으로 오염됩니다.
### 5. 네임스페이스(Namespace) 확인 기법

- locals() : 현재 코드가 실행 중인 지역 영역(Local)의 변수와 주소 정보들을 딕셔너리 형태로 반환합니다.
- globals() : 파일 전체에서 접근 가능한 전역 영역(Global)의 변수, 함수, 임포트된 모듈 정보들을 반환합니다.
- dir(__builtins__) : 파이썬이 아무런 임포트 없이 기본적으로 들고 태어난 모든 내장 함수와 에러 이름들을 리스트 형태로 열람합니다. (예: print, max, len 등)
### 6. [★주의] 인자 주입의 철칙: 위치 인자(Positional) vs 키워드 인자(Keyword)

- 기본 약속 : 인자를 순서대로 쑤셔 넣는 것을 위치 인자, 변수명 = 값 형태로 콕 집어 넣는 것을 키워드 인자라고 부릅니다.
- 철칙: 위치 인자가 무조건 먼저 와야 함 : 호출 시 키워드 인자를 한 번이라도 선언했다면, 그 뒤로는 절대 위치 인자가 따라올 수 없습니다. (SyntaxError 대폭발)
### 7. 디폴트 파라미터(Default Parameter)의 배치 제약

- 철칙: 디폴트 파라미터는 무조건 뒤쪽으로 몰아야 함 : 인자 값이 안 들어왔을 때 기본값을 채워주는 디폴트 인자(b = 200) 뒤에는 절대 기본값이 없는 순수 위치 파라미터가 올 수 없습니다. ([6-13])
### 8. 강제 키워드 지정 파라미터 (Keyword-Only Parameter)

- 별표* 단독 기호의 의미 : 함수를 정의할 때 매개변수 사이에 별표*를 단독으로 끼워 넣으면, "이 별표 뒷방에 있는 변수들은 순서대로 넣을 생각 추호도 하지 말고, 무조건 이름 = 값 형태로만 대입해라" 하고 강제하는 벽 역할을 합니다.
- 실무적 가치 : 함수에 인자가 너무 많아서 순서가 헷갈려 실수가 터질 것 같거나, 특정 옵션 인자를 개발자가 명확하게 인지하고 쓰게 만들고 싶을 때 안전장치로 사용합니다.
```python
# [6-14] 별표 뒤의 a, b, c는 오직 키워드로만 주입 가능
def func(*, a, b = 200, c):
    print(a, b, c)

func(a = 10, c = 30) # 성공 (b는 디폴트값 200으로 자동 충돌 우회)
func(10, 20, 30)     # ⚠️ 실패 (TypeError: 위치 인자 허용 안 함)


# [6-15] 응용 조합 (a, b는 순서대로 가능 / c, d, e는 무조건 키워드로만)
def func(a, b = 200, *, c, d = 400, e):
    pass

func(10, e = 50, c = 30) # 성공 (b와 d는 알아서 기본값 채워짐)
```

### 💡 부족했던 디테일 한 스푼 보정 (추가/수정 정보)

1. 가변 인자(args)와의 차이 인지 : 이 단원에서 나온 별표 단독 기호*는 변수명 없이 혼자 쓰이는 특수 문법입니다. 나중에 배우게 될 def func(*args) 처럼 뒤에 변수 이름이 붙는 '가변 인자(여러 개를 튜플로 한 통에 받는 기술)'와 모양이 비슷해서 헷갈리기 쉬우니 "독립된 별표 쉼표(, )는 뒤쪽 인자들을 키워드로 잠그는 빗장이다" 라고 명확히 구별해 묶어두셔야 합니다.
