# 💻 RISC vs CISC 구조 총정리

## 🌓 1. RISC와 CISC 한눈에 비교

| 구분 | RISC (Reduced Instruction Set Computer) | CISC (Complex Instruction Set Computer) |
|---|---|---|
| 개념 | 단순하고 필수적인 명령어 세트만 사용 | 복잡하고 다양한 기능의 명령어 세트 사용 |
| 명령어 길이 | 고정 폭 (모든 명령어의 크기가 일정함) | 가변 폭 (명령어마다 크기가 다름) |
| 동작 속도 | 명령어당 1클럭 내외로 처리 (매우 빠름) | 명령어마다 처리 클럭이 다름 (느림) |
| 설계 구조 | 하드웨어가 단순하고 컴파일러가 복잡함 | 하드웨어가 복잡하고 컴파일러가 단순함 |
| 메모리 접근 | 오직 Load/Store 명령어로만 메모리 접근 가능 | 명령어 자체에서 직접 메모리 접근 가능 |
| 소비 전력 | 매우 적음 (초절전) | 많음 (발열 및 전력 소모 큼) |
| 대표 아키텍처 | ARM (STM32, 스마트폰 AP), RISC-V | Intel x86 / AMD (데스크톱, 노트북 PC) |

## 🏎️ 2. RISC (축소 명령어 집합 컴퓨터)

### 🟢 장점

- 고속 처리 (파이프라이닝 최적화): 모든 명령어의 길이가 같고 단순해서 한 번에 여러 명령어를 겹쳐서 처리하는 '파이프라인' 기술을 쓰기 매우 유리합니다.
- 낮은 전력 소모와 적은 발열: 회로가 단순해서 칩 크기도 작고 전기를 아주 적게 먹습니다. (배터리로 구동해야 하는 스마트폰이나 임베디드 기기에 절대적인 이유)
- 저렴한 칩 생산 비용: 반도체 설계 및 제조가 비교적 쉬워 단가가 낮습니다.
- 컴파일러 의존도 높음: 하드웨어가 하던 복잡한 최적화 작업을 소프트웨어(컴파일러)가 대신 해줘야 하므로 정교한 컴파일러 기술이 필요합니다.
- 프로그램 코드 크기 증가(성능 저하): 명령어 하나가 단순한 일만 하므로, 복잡한 기능을 구현하려면 여러 개의 명령어를 조합해서 코딩해야 합니다. 결과적으로 컴파일된 바이너리 크기가 커집니다.
## 🏢 3. CISC (복잡 명령어 집합 컴퓨터)

### 🟢 장점

- 짧은 코드 크기: 명령어 하나가 매우 복잡하고 강력한 연산(예: 메모리에서 값을 가져와 더하고 다시 메모리에 쓰는 작업을 명령어 한 줄로 처리)을 수행하므로 전체 프로그램 코드 크기가 작아집니다.
- 컴파일러 설계 용이: 하드웨어가 웬만한 복잡한 기능을 명령어로 다 지원해주므로, 컴파일러가 골치 아프게 명령어를 쪼갤 필요가 없습니다.
- 메모리 절약: 옛날처럼 메모리 가격이 비싸던 시절에 프로그램 크기를 최소화할 수 있는 강력한 장점이었습니다.
### 🔴 단점

- 복잡하고 거대한 회로: 다양한 명령어를 하드웨어적으로 다 지원해야 해서 내부 회로가 엄청나게 복잡하고 칩 크기가 큽니다.
- 높은 발열과 전력 소모: 복잡한 만큼 전기를 많이 먹고 뜨겁습니다. (냉각팬이 필수인 이유)
- 비효율적인 성능 극대화: 명령어들의 길이가 제각각이라 고속으로 처리하기 위한 내부 구조(파이프라인) 설계가 까다롭습니다.
## 💡 임베디드 개발자 실무 한마디

우리가 공부하는 STM32 M4 칩(ARM Cortex-M4 아키텍처)은 대표적인 RISC 기반입니다.

따라서 STM32로 코드를 짜고 디버깅을 할 때 디스어셈블리(어셈블리어 코드) 창을 열어보면, 복잡한 계산식 하나가 LDR(메모리 읽기), ADD(더하기), STR(메모리 쓰기) 처럼 아주 잘게 쪼개진 단순 명령어들의 나열로 변환되는 것을 볼 수 있습니다. 이 과정이 바로 RISC가 작동하는 본질적인 모습입니다!

# 🚀 오픈소스 프로세서, RISC-V란 무엇일까?

> 한 줄 요약 (안드로이드 vs iOS 비유)

## 🌓 1. RISC-V의 탄생과 핵심 개념

- 개념: 2010년 미국 UC 버클리 대학에서 시작된 오픈소스 기반의 32/64비트 RISC 명령어 집합 구조(ISA)입니다.
- ISA란? CPU 하드웨어가 소프트웨어의 명령(C언어 코드 등)을 알아듣기 위한 일종의 '사전(규격)'입니다.
- 오픈소스의 의미: 기존 ARM이나 Intel x86 아키텍처를 쓰려면 천문학적인 라이선스 비용과 로열티를 내야 했습니다. 하지만 RISC-V 규격은 전 세계 누구나 무료(로열티 0원)로 가져다 직접 칩을 설계하고 판매할 수 있습니다.
## 🛠️ 2. RISC-V의 독보적인 강점

### ① 완벽한 설계 자유도 (레고 블록 방식)

- 종합 세트처럼 안 쓰는 기능까지 무겁게 다 들어가 있는 기존 CPU와 다릅니다.
- '기본 정수 연산 뼈대(Base)'에다가 내가 필요한 기능(곱셈, 실수 연산, AI 가속용 벡터 연산 등)만 확장팩(Extension) 형태로 골라서 조립하듯 칩을 설계할 수 있습니다.
- 덕분에 칩의 크기와 전력 소모를 극한으로 줄일 수 있습니다.
### ② 라이선스 비용 Zero

- 스타트업이나 중소기업도 특허 소송 걱정 없이 고성능 CPU 코어를 설계할 수 있게 되었습니다.
- 실제로 삼성을 비롯해 구글, NVIDIA, 애플, 인텔 등 빅테크 기업들도 내부 제어용 칩이나 AI 가속기에 RISC-V를 적극적으로 도입해 비용을 대폭 아끼고 있습니다.
### ③ 반도체 공급망 리스크 탈피 (반도체 자립)

- 특정 국가(미국, 영국 등)가 기술 수출 제약을 걸더라도, 비영리 재단이 관리하는 글로벌 개방형 표준이기 때문에 정치적 리스크에 휘둘리지 않고 독자적으로 반도체 생태계를 구축할 수 있습니다.
## ⚠️ 3. RISC-V의 현실적인 한계점 (ARM을 당장 못 넘는 이유)

- 소프트웨어 생태계의 파편화: 누구나 설계를 마음대로 바꿀 수 있다 보니, 칩마다 들어간 명령어셋이 제각각이 되기 쉽습니다. 이는 소프트웨어 개발자가 동일한 프로그램을 모든 RISC-V 칩에서 똑같이 실행하기 어렵게 만드는 단점이 됩니다.
- 아직은 부족한 인프라: ARM은 수십 년간 쌓여온 막강한 개발 툴, 디버거, 컴파일러, OS(윈도우, 안드로이드 등) 생태계가 완벽하게 무장되어 있어 개발이 편합니다. RISC-V는 생태계가 빠르게 크고 있지만 여전히 갈 길이 멉니다.
## 💡 임베디드 시선에서의 전망

최근에는 STM32의 제조사인 STMicroelectronics나 ESP32를 만드는 Espressif 같은 마이크로컨트롤러(MCU) 대기업들도 RISC-V 기반 코어를 탑재한 초저전력 MCU들을 공격적으로 출시하고 있습니다.

앞으로 임베디드 엔지니어라면 ARM(Cortex-M)뿐만 아니라 이 RISC-V 기반 칩들의 개발 환경도 반드시 마주치게 될 것입니다.

# 👑 [ARM 아키텍처] Cortex Core 및 Profile(A·R·M·X) 핵심 정리

## 🌓 1. Cortex-A, R, M, X 한눈에 비교

| 구분 | Cortex-X | Cortex-A (Application) | Cortex-R (Real-time) | Cortex-M (Microcontroller) |
|---|---|---|---|---|
| 타겟 장치 | 초고성능 스마트폰, 노트북, AI 서버 | 스마트폰, 태블릿, 싱글보드 컴퓨터(라즈베리파이) | 자동차 제어기(ABS, ECU), SSD 컨트롤러, 5G 모뎀 | MCU (STM32, 스마트폰 센서 허브, 소형 디바이스) |
| 운영체제(OS) | Windows, macOS, 고성능 Linux | Android, iOS, Full Linux | 실시간 운영체제(RTOS) 또는 No OS | RTOS (FreeRTOS) 또는 No OS (Bare-metal) |
| 핵심 하드웨어 | 극대화된 IPC 성능, 대용량 캐시 | MMU (가상 메모리 관리 장치) | MPU (메모리 보호 장치), 듀얼 코어 록스텝 | MPU (옵션), NVIC (빠른 인터럽트 컨트롤러) |
| 핵심 가치 | 절대적 단일 코어 성능 극대화 | 멀티태스킹, 가상 메모리 관리 | 예측 가능성 (Deter-ministic), 초고속 실시간 반응 | 초저전력, 저렴한 가격, 뛰어난 가성비 |

- 목표: 전력 소모를 타협하더라도 경쟁사(Apple, Intel 등)를 압도할 수 있는 절대적인 싱글 코어 성능을 뽑아내는 것.
- 특징: 원래 ARM은 저전력이 무기였으나, 플래그십 스마트폰과 AI 노트북 시장을 위해 특별히 설계된 초고성능 코어입니다. 스마트폰 AP(스냅드래곤, 엑시노스 등)에서 가장 무거운 연산을 처리하는 메인 코어(빅코어)로 탑재됩니다.

# ⚙️ [핵심 프로세서] CPU, FPU, GPU, Coprocessor 역할 요약

- CPU (중앙 처리 장치): 컴퓨터의 핵심 메인 두뇌입니다. 주로 정수 연산, 시스템 제어, 논리 판단 등 전반적인 명령 처리를 총괄합니다.
- FPU (부동소수점 연산 장치): 소수점이 포함된 복잡한 실수 연산을 전문적으로 빠르게 처리하는 장치입니다. (STM32 M4에도 FPU가 내장되어 있어 센서 필터링 등의 소수점 연산 속도가 매우 빠릅니다.)
- GPU (그래픽 처리 장치): 화면에 이미지를 그리거나, 수천 개의 단순 반복 연산(행렬 연산, AI 연산 등)을 동시에 병렬로 처리하는 장치입니다.
- Coprocessor (보조 프로세서): CPU의 연산 부담을 덜어주기 위해 특정 기능만 전담하여 도와주는 도우미 프로세서입니다. (위에 언급한 FPU나 암호화 칩, 사운드 가속기 등이 모두 대표적인 코프로세서에 해당합니다.
제어할 대상이 하드웨어(GPIO, I2C)든 기억장치(RAM, ROM)든, 내부 레지스터를 가지고 있어 **주소(Address)를 할당받을 수 있다면 CPU에게는 전부 똑같은 '메모리'**로 보입니다.

- 32비트 공간 = 2^32 = 4,294,967,296 = 4,096MB = 4GB (주소 범위: 0x00000000 ~ 0xFFFFFFFF)
- 512KB 공간 = 2^19 = 524,288B = 19비트 주소선 (주소 범위: 0x00000 ~ 0x7FFFF)
- 128KB 공간 = 2^17 = 131,072B = 17비트 주소선 (주소 범위: 0x00000 ~ 0x1FFFF)
- 32KB 공간 = 2^15 = 32,768B = 15비트 주소선 (주소 범위: 0x0000 ~ 0x7FFF)
- 8KB 공간 = 2^13 = 8,192B = 13비트 주소선 (주소 범위: 0x0000 ~ 0x1FFF)
- 256B 공간 = 2^8 = 256B = 8비트 주소선 (주소 범위: 0x00 ~ 0xFF)
- 2^10 = 1KB / 2^20 = 1MB / 2^30 = 1GB / 2^40 = 1TB
