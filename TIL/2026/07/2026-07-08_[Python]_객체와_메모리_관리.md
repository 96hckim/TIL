# 🏗️ [Chapter 2] 파이썬의 객체 구조와 메모리 관리

### [2-1] 파이썬의 모든 것은 객체다

- 파이썬에서는 숫자, 문자, 심지어 데이터 타입(int, str 등) 자체까지도 모두 객체(Object)입니다.
- int는 정수를 만드는 클래스(붕어빵 틀)이며, int(10)이나 그냥 숫자 10은 클래스를 통해 만들어진 인스턴스(진짜 붕어빵)입니다.
```python
print(int, type(int)) # int 타입 자체도 <class 'type'>이라는 객체입니다.

x = int(10)
print(x, type(x)) # x는 int 클래스의 인스턴스 (<class 'int'>)

y = int('20')
print(y, type(y)) # 문자열을 숫자로 바꿔 인스턴스 생성

print(10, type(10)) # 변수에 담지 않은 일반 숫자도 내부에선 int 인스턴스입니다.
```

### [2-2] 객체 분석을 위한 내장 함수 (dir, id, isinstance)

- dir(): 객체가 가지고 있는 모든 변수와 메서드(함수)의 목록을 확인합니다.
- id(): 객체가 저장된 고유한 메모리 주소값을 반환합니다.
- isinstance(인스턴스, 클래스): 특정 객체가 그 클래스로 만들어진 인스턴스가 맞는지 검사하여 True/False를 반환합니다.
```python
print(dir(int), dir(3), sep="\n") # int 클래스와 숫자 3이 가진 기능(메서드) 목록 출력
print(id(int), id(3))             # 각각의 고유 메모리 주소 확인

print(isinstance(3, int))     # True (숫자 3은 정수 인스턴스가 맞음)
print(isinstance(int, int))   # False (int 자체는 정수 인스턴스가 아님)
print(isinstance(int, type))  # True (int 클래스는 'type'이라는 설계도로 만든 것)

a = 3
print(a == 3, a != 3) # 값 비교 연산
```

### [2-3] 클래스 메서드와 인스턴스 메서드 호출 방식

- 파이썬 내부에서는 우리가 쓰는 더하기(+)나 대문자 변환(.upper())이 모두 특정 메서드를 호출하는 방식으로 돌아갑니다.
- 클래스 기준 호출: 클래스명.메서드(인스턴스, 값) 형태로 원형 그대로 호출할 수 있습니다.
- 인스턴스 기준 호출: 인스턴스.메서드(값) 형태로 주체를 앞에 두고 호출하며, 실무에서 주로 쓰는 방식입니다.
```python
# 1. 덧셈 원리 (__add__ 매직 메서드)
print(int.__add__(3, 4)) # 클래스에서 호출: 3과 4를 더함 (7)
x = int(3)
print(x.__add__(4))      # 인스턴스(x)에서 호출: 내 값(3)에 4를 더함 (7)
print((3).__add__(4))    # 숫자 3 자체도 객체이므로 바로 메서드 호출 가능 (7)

# 2. 대문자 변환 원리
print(str.upper("Hello")) # 클래스에서 호출하여 "Hello"를 전달 -> "HELLO"
x = str("Hello")
print(x.upper())          # 인스턴스(x) 자체에서 호출 -> "HELLO"
print("Hello".upper())    # 문자열 상수에서 바로 호출 -> "HELLO"
```

### [2-4] 연산자 오버로딩 (Operator Overloading)

- 우리가 편하게 쓰는 연산자(+, )나 내장 함수(abs, round)는 사실 내부적으로 숨겨진 매직 메서드(Magic Method, 앞뒤로 __가 붙은 메서드)를 실행하는 껍데기(단축키)에 불과합니다.
```python
# [A] 우리가 작성하는 일반적인 코드
a = 1 + 2
b = "x" + "y"
c = 3.4 - 2.5
d = abs(-10)
e = abs(-3.8)
f = round(1234, -2)   # 십의 자리에서 반올림하여 백의 자리까지 표시 (1200)
g = round(10.434, 1)  # 소수점 둘째 자리에서 반올림하여 첫째 자리까지 표시 (10.4)

print(a, b, c, d, e, f, g, sep="\n")

# [B] 파이썬 내부에서 실제로 변환되어 실행되는 매직 메서드 형태 (위와 100% 동일)
a = int.__add__(1, 2)
b = str.__add__("x", "y")
c = float.__sub__(3.4, 2.5)
d = int.__abs__(-10)
e = float.__abs__(-3.8)
f = int.__round__(1234, -2)
g = float.__round__(10.434, 1)

print(a, b, c, d, e, f, g, sep="\n")
```

## 3. 메모리 관리와 이름 바인딩 (Name Binding)

### [2-5] 이름 바인딩과 메모리 주소 공유

- 파이썬은 변수에 값이 직접 저장되는 것이 아니라, 메모리에 저장된 값의 주소를 변수가 가리키는(참조, Binding) 방식입니다.
- 동일한 값을 여러 변수에 할당하면, 메모리 낭비를 막기 위해 같은 주소를 가리키게 됩니다.
```python
a = 2020
b = 2020
# 2020이라는 객체의 주소, a의 주소, b의 주소가 모두 동일합니다.
print(id(2020), id(a), id(b))
```

### [2-6] 네임스페이스(Name space) 추적 및 재할당

- 변수의 값을 바꾸면 기존 메모리에 있던 값이 수정되는 것이 아니라, 새로운 연산 결과 객체가 메모리에 만들어지고 변수가 가리키는 주소가 바뀝니다.
- globals() 함수를 사용하면 현재 프로그램의 전역 공간(Namespace)에 어떤 변수들과 객체들이 매칭되어 있는지 딕셔너리 형태로 훔쳐볼 수 있습니다.
```python
a = 10
print(id(a)) # 처음 숫자 10의 메모리 주소

a = a + 1    # 10 + 1 계산 결과인 '11' 객체가 새로 생성되고 a는 11을 가리킴
b = 10       # b는 기존에 존재하던 '10' 객체의 주소를 가리킴
print(a, b)  # 11, 10

print(id(a)) # 11의 주소로 바뀜 (처음 주소와 다름)
print(id(b)) # b는 기존 10의 주소를 가져옴 (a의 처음 주소와 같음)

# 현재 등록된 전역 변수와 객체들의 매칭 상태(네임스페이스)를 출력
print(globals())
```

### [2-7] 복합 대입 연산자 (Augmented Assignment)와 메모리 주소

- +=, =, = 와 같이 산술 연산과 대입을 동시에 하는 연산자입니다.
- 파이썬의 정수(int)는 불변(Immutable) 객체이므로, 복합 대입 연산자를 쓸 때마다 매번 새로운 연산 결과 객체가 메모리에 생성되면서 변수의 주소(id)가 계속 바뀝니다.
```python
a = 20
b = 4

print(id(a), a)
a += 1          # a = a + 1 과 동일. 21이 새로 생성되며 주소 변경
print(id(a), a)
a -= b          # a = a - b 와 동일. 17이 새로 생성되며 주소 변경
print(id(a), a)
a *= b + 2      # a = a * (b + 2) 와 동일. 우변이 먼저 계산된 후 연산 수행 (17 * 6 = 102)
print(id(a), a) # 102가 새로 생성되며 주소 변경
```

### [2-8] 이름 충돌 문제 (Name Resolution)

- 파이썬은 변수 이름을 찾을 때 내가 만든 이름(Local/Global) ➡️ 파이썬 내장 이름(Built-in) 순서로 검색합니다.
- 만약 파이썬의 기본 내장 함수 이름(id, int 등)을 변수명으로 사용해 버리면, 기존 내장 함수 기능을 덮어써 버려 더 이상 함수로 호출할 수 없는 문제가 발생합니다.
```python
x = id(10)
print(x)

id, int = 10, 20 # ⚠️ 위험! 내장 함수(id, int)의 이름을 정수형 변수로 덮어씀
print(id, int)   # 출력: 10 20 (이제 id와 int는 그냥 변수일 뿐입니다.)

# print(id(10))  # ❌ 에러 발생! 변수 'id'는 10이 담겨있으므로 함수처럼 호출하면 TypeError가 납니다.
```

### [2-9] 파이썬 키워드(Keyword) 확인

- 파이썬 내부 문법을 위해 이미 예약된 단어(예: False, None, if, for, import 등) 목록입니다.
- 이 단어들은 강력한 규칙으로 보호받고 있어서, [2-8]과 다르게 아예 변수명으로 사용하는 것이 문법적으로 불가능하며 시도 시 SyntaxError가 납니다.
```python
import keyword
print(keyword.kwlist) # 파이썬에서 절대 변수명으로 쓸 수 없는 키워드 목록 인쇄
```

### [2-10] 변수 이름 관련 에러 (NameError)

- 프로그램에 존재하지 않는 변수 이름을 사용하려고 할 때 발생하는 에러입니다.
- 정의되지 않은 변수(y)를 대입하려 하거나, 선언하지 않은 변수(a)에 값을 더하려 할 때 발생합니다.
```python
# x = y  # ⚠️ NameError: 정의된 적 없는 'y'를 불러와서 발생
# a += 1 # ⚠️ NameError: 'a'라는 상자가 아직 없는데 1을 더하라고 해서 발생
```

### [2-11] 네임스페이스의 이름 삭제 (del)

- del 변수명을 사용하면 메모리 공간 내에서 해당 변수(이름)를 깨끗하게 지울 수 있습니다.
- 이를 응용하면 [2-8]에서 실수로 덮어썼던 내장 함수 이름(id, int)을 지워서 다시 파이썬 원래의 내장 함수 기능으로 복구시킬 수 있습니다.
```python
x = id(10)
print(x)
id, int = 10, 20 # 덮어쓰기

del id   # 내가 새로 만든 전역 변수 'id'를 네임스페이스에서 삭제
del int  # 내가 새로 만든 전역 변수 'int'를 네임스페이스에서 삭제

print(id(10)) # ✨ 복구 성공! 전역 변수가 사라져서 파이썬 원래의 내장 id() 함수가 호출됨

del x
# print(x) # ⚠️ 에러 발생! 이미 지워진 변수 'x'를 불러왔기 때문

# del float # ⚠️ 내장 함수 자체는 del로 직접 파괴할 수 없어 무시되거나 에러가 납니다.
```
