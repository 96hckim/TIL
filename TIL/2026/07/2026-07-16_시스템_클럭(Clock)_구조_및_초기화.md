### 1. PLL이 왜 필요한가요? (하드웨어적 한계)

STM32 같은 MCU가 168MHz나 84MHz 같은 고속으로 생생 돌려면 외부에서 그만큼 빠른 속도로 틱(Tick)을 넣어줘야 할 것 같습니다. 하지만 현실은 그렇지 못합니다.

- 외부 크리스탈(HSE)의 한계: 칩 밖에 달아주는 정밀한 돌멩이(크리스탈 Oscillator)는 보통 8MHz나 25MHz 수준입니다. 이걸 넘어가서 100MHz가 넘는 크리스탈을 만들려고 하면 가격이 폭등하고, 무엇보다 노이즈가 너무 심해져서 회로가 망가집니다.
- 해결책: "밖에서는 조용하고 안정적인 8MHz짜리 저속 클럭을 받아오자. 그리고 칩 내부에서 PLL이라는 뻥튀기 기계에 통과시켜 168MHz로 속도를 곱해(체배) 사용하자!"
### 2. PLL의 작동 원리 (주파수 뻥튀기 3단계)

STM32 클럭 설정 화면(Clock Configuration)을 보면 복잡한 공식이 나오는데, 원리는 단순한 곱셈과 나눗셈입니다.

```plain text
[입력: 외부 8MHz] ➡️ [/ M (나눗셈)] ➡️ [ x N (VCO 곱셈)] ➡️ [/ P (나눗셈)] ➡️ [출력: 시스템 168MHz]
```

1. 분주기 (Prescaler, /M): 들어온 8MHz를 일단 작게 쪼갭니다. (예: 8MHz / 8 = 1MHz)
1. 배율기 (VCO, xN): 쪼개진 깨끗한 1MHz 신호를 수백 배로 뻥튀기합니다. (예: 1MHz x 336 = 336MHz)
1. 출력 분주기 (Postscaler, /P): 너무 커진 속도를 시스템 최대 한계에 맞춰 다시 적절히 나눕니다. (예: 336MHz / 2 = 168MHz)
이렇게 외부의 흔하디흔한 8MHz짜리 신호가 PLL 기어박스를 거치면서 오차 없이 깨끗한 168MHz의 초고속 클럭으로 변신합니다.

### ⚠️ 실무에서 기억해야 할 '주의점'

PLL을 만질 때 반드시 알아야 할 하드웨어 규칙이 있습니다.

- 동작하는 데 시간이 걸립니다 (Lock Time):
### 💡 3줄 요약

1. PLL은 외부의 느리고 안정적인 클럭(8MHz)을 받아서 내부에서 초고속 클럭(168MHz)으로 뻥튀기해 주는 주파수 곱셈기이다.
1. 내부적으로는 나누고(/M) ➡️ 곱하고(xN) ➡️ 다시 나누는(/P) 공식을 거쳐 원하는 속도를 정밀하게 조율한다.
1. 켜자마자 바로 쓸 수 있는 게 아니라, 주파수가 고정될 때까지 기다리는 대기 시간(Lock Time)이 필요하다.

# [RCC]

## 1. 레지스터 핵심 역할

- RCC->CR: 클럭 전원 스위치 & 안정화(Ready) 확인
- FLASH->ACR: 느린 Flash 메모리 대기 시간(Wait State) & 가속 캐시 설정
- RCC->PLLCFGR: PLL 곱셈/나눗셈 기어박스 설정 (M, N, P, Q)
- RCC->CFGR: 내부 버스 분주비 설정 & 메인 클럭 최종 스위칭
## 2. 핵심 버스 클럭 계층도

| 클럭 이름 | 전체 이름 | 실제 역할 및 최대 스펙 | 우리의 설정값 |
|---|---|---|---|
| SYSCLK | System Clock | 시스템 전체의 심장 (최대 100MHz) | 96MHz |
| HCLK | AHB Clock | High-speed 버스 (CPU 코어, Flash, SRAM, DMA) | 96MHz (1분주) |
| PCLK2 | APB2 Clock | 고속 주변장치 버스 (SPI, ADC 등 / 최대 100MHz) | 96MHz (1분주) |
| PCLK1 | APB1 Clock | 저속 주변장치 버스 (UART, I2C, CAN 등) | 48MHz (2분주) |

## 3. PLL 설정 공식 및 변수 (입력: HSI 16MHz)

### 3대 핵심 공식

f_{VCO} = \text{입력 클럭(16MHz)} \times \frac{PLLN}{PLLM}

SYSCLK = \frac{f_{VCO}}{PLLP}

USBCLK = \frac{f_{VCO}}{PLLQ}

### 실제 적용된 파라미터 값

- PLLSRC: 0 (입력 소스로 내부 HSI 16MHz 선택)
- PLLM: 8 (16MHz / 8 = 2MHz로 쪼개어 VCO 내부 진입)
- PLLN: 192 (2MHz * 192 = 384MHz로 뻥튀기)
- PLLP: 4 (레지스터 설정값 1 ➡️ 384\text{MHz} / 4 = \mathbf{96\text{MHz}} 확보)
- PLLQ: 8 (384\text{MHz} / 8 = \mathbf{48\text{MHz}}로 USB 전용 클럭 확보)
## 4. Flash Memory와 Wait State (대기 주기)

### 💡 왜 대기 주기가 필요한가?

- 원인: CPU는 96MHz(매우 빠름)로 도는데, Flash 메모리는 물리적으로 약 30ns(상대적으로 느림)의 읽기 속도 한계를 가집니다.
- 결과: 대기 없이 무작정 읽으면 쓰레기 값을 읽어 HardFault(시스템 다운)가 발생합니다.
- 해결책: CPU가 데이터가 준비될 때까지 기다리도록 Wait State를 줍니다.
### 96MHz 기준 Flash 설정 (FLASH->ACR)

- Wait State (Latency): 96MHz가 속한 영역(90 ~ 120MHz)이므로 3 Wait State 설정 (0x3)
- 가속 장치 활성화: 대기로 인한 속도 저하를 막기 위해 Instruction Cache(ICEN), Data Cache(DCEN), Prefetch(PRFTEN) 가속 기능을 모두 활성화합니다 (FLASH->ACR 상위 비트 세팅).
## 5. Clock_Init() 6단계 실행 시퀀스

클럭을 전환할 때는 하드웨어가 뻗지 않도록 반드시 아래의 정해진 순서를 지켜야 합니다.

> [1] HSI On ➡️ [2] Flash Latency(3WS) 선행 설정 ➡️ [3] PLL 파라미터 등록 ➡️ [4] PLL On & Lock 대기 ➡️ [5] 버스 분주비 세팅 ➡️ [6] SYSCLK 소스 PLL로 전환

```c
void Clock_Init(void)
{
    /* ====================================================================
     * [1] RCC->CR 제어: 내부 고속 클럭(HSI 16MHz) 활성화 및 대기
     * ==================================================================== */
    RCC->CR |= (1 << 0);                     // Bit 0 (HSION): HSI 클럭 전원 켜기
    while(!Macro_Check_Bit_Set(RCC->CR, 1)); // Bit 1 (HSIRDY): HSI 발진기가 물리적으로 안정화(Ready)될 때까지 대기


    /* ====================================================================
     * [2] FLASH->ACR 제어: CPU 초고속(96MHz) 전환 대비 대기 주기 및 캐시 설정
     * ==================================================================== */
    // 캐시 리셋(비우기)을 통해 오동작 방지
    FLASH->ACR = (1 << 12) |                 // Bit 12 (ICRST): Instruction Cache(명령 캐시) 초기화 리셋
                 (1 << 11);                  // Bit 11 (DCRST): Data Cache(데이터 캐시) 초기화 리셋
    
    // 캐시 가속기 활성화 및 대기 사이클(Wait State) 적용
    FLASH->ACR = (1 << 10) |                 // Bit 10 (DCEN): 데이터 캐시(Data Cache) 가속기 활성화
                 (1 << 9)  |                 // Bit 9 (ICEN): 명령 캐시(Instruction Cache) 가속기 활성화
                 (1 << 8)  |                 // Bit 8 (PRFTEN): 프리페치(Prefetch) 버퍼 미리 읽기 가속 활성화
                 (0x3 << 0);                 // Bits 2:0 (LATENCY): 3 Wait States 설정 (96MHz 속도의 지연 보쇄용)


    /* ====================================================================
     * [3] RCC->PLLCFGR 제어: 메인 주파수 조율 엔진(PLL) 기어 세팅
     * ==================================================================== */
    RCC->PLLCFGR = (8 << 24)  |  // [PLLQ]   USB/SDIO용 분주 기어 (Bits 27:24) -> 384MHz / 8 = 48MHz
                   (0 << 22)  |  // [PLLSRC] PLL 입력 소스 선택 (Bit 22) -> 0: HSI, 1: HSE
                   (1 << 16)  |  // [PLLP]   SYSCLK용 분주 기어 (Bits 17:16) -> 384MHz / 4 = 96MHz (값 1은 4분주)
                   (192 << 6) |  // [PLLN]   VCO 증폭 곱셈 기어 (Bits 14:6) -> 2MHz * 192 = 384MHz
                   (8 << 0);     // [PLLM]   입력 주파수 1차 감쇄 기어 (Bits 5:0) -> HSI 16MHz / 8 = 2MHz


    /* ====================================================================
     * [4] RCC->CR 제어: 세팅 완료된 PLL 엔진 가동 및 대기
     * ==================================================================== */
    Macro_Set_Bit(RCC->CR, 24);              // Bit 24 (PLLON): PLL 메인 엔진 가동 시작
    while(!Macro_Check_Bit_Set(RCC->CR, 25)); // Bit 25 (PLLRDY): PLL 주파수 고정(Lock/Ready) 완료 대기


    /* ====================================================================
     * [5] RCC->CFGR 제어: 내부 버스 클럭 분주비 설정 (시스템 폭주 전 방어선 형성)
     * ==================================================================== */
    RCC->CFGR = (0 << 13) |  // [PPRE2] APB2 고속 버스 분주 설정 (Bits 15:13) -> 0xx: 1분주 (96MHz)
                (4 << 10) |  // [PPRE1] APB1 저속 버스 분주 설정 (Bits 12:10) -> 100: 2분주 (48MHz, 최대 50MHz 제약 준수)
                (0 << 4);    // [HPRE]  AHB 버스 분주 설정 (Bits 7:4)   -> 0xxx: 1분주 (96MHz)


    /* ====================================================================
     * [6] RCC->CFGR 제어: 시스템 메인 클럭(SYSCLK)을 PLL로 최종 전환 및 검증
     * ==================================================================== */
    Macro_Write_Block(RCC->CFGR, 0x3, 0x2, 0); // Bits 1:0 (SW): 시스템 클럭 소스 선택 -> 0x2(10) 대입으로 PLL 선택
    while(Macro_Extract_Area(RCC->CFGR, 0x3, 2) != 0x2); // Bits 3:2 (SWS): 시스템 클럭 소스 상태 -> PLL(0x2) 전환 완료 대기
}
```

# [Macro_Set_Bit(RCC->AHB1ENR, 0);]

### 1. 무슨 뜻인가요?

- RCC->AHB1ENR: AHB1 버스에 연결된 주변장치들의 클럭(전원)을 켜고 끄는 레지스터입니다. (AHB1 Peripheral Clock Enable Register)
- 0: 이 레지스터의 0번째 비트를 의미하며, 이 자리가 바로 GPIOAEN (GPIOA 클럭 활성화) 비트입니다.
- Macro_Set_Bit(..., 0): 0번째 비트를 1로 세트하여 전원을 켭니다.
### 2. 왜 하나요?

STM32를 포함한 ARM MCU들은 전력 소모를 최소화하기 위해 처음 켜질 때 모든 주변장치의 전원(클럭)을 꺼둡니다.

따라서 GPIOA에 연결된 핀(예: LED, 버튼 등)을 제어하기 전에 반드시 이 코드로 "나 이제 GPIOA 포트 쓸 거니까 전원 켜줘!" 하고 깨워야 합니다. 이 코드를 생략하면 GPIOA 레지스터에 무슨 값을 써도 전혀 동작하지 않습니다.
