### 🛠️ [Chapter 10]

- .append(x) vs .extend(Iterable) ([10-1]) : .append()는 인자를 묶음째로 들고 가 한 개의 방(Slot)을 확장하지만, .extend()는 인자의 껍데기를 한 풀 벗겨내어 원소별로 방을 쪼개 개별 안착시킵니다. (튜플 주입 시 가공 형태 유의)
- .remove(x)의 원소 탐색 한계 : 리스트 내부에서 지정된 값(x)과 == 동등성이 성립하는 최초의 첫 번째 데이터만 선착순으로 삭제하며, 일치하는 값이 아예 없을 경우 ValueError를 발생시켜 스크립트를 중단시킵니다.
- .pop(index)의 자원 회수 법칙 : 지정한 인덱스 방의 알맹이를 메모리 상에서 파괴하는 동시에 그 값을 바깥으로 반환(Return)해 줍니다. 인덱스를 생략하면 맨 뒷방(1)을 자동 회수하며, 원본 장부의 전체 길이가 실시간으로 줄어듭니다.
- .reverse()와 .sort()의 인플레이스(In-place) 제어 : 새 객체를 복사하지 않고 원본 리스트 주소판 내부에서 직접 순서를 뒤집거나 정렬(Timsort 알고리즘)하여 구조를 영구 변형합니다. 두 메서드 모두 연산 종료 후 None을 반환하므로, l = l.sort() 처럼 대입 연산을 시도하면 장부 전체가 날아가는 대참사가 발생합니다.
```python
# [10-1] append와 extend의 자료형 징집 차이 복습
l = [1]
l.append((3, "KIM"))  # l 결과: [1, (3, 'KIM')] -> 하나의 방에 튜플 통째로 주입
l.extend((6, "LEW"))  # l 결과: [1, (3, 'KIM'), 6, 'LEW'] -> 튜플 껍데기를 파괴하고 원소만 삽입
```

- 순서 보존식 중간 삽입 ([10-2]) : 이미 오름차순으로 정렬된 장부에 새로운 데이터 m을 밀어 넣을 때, for-else 만기 졸업 메커니즘을 결합하면 sort() 함수 없이 완벽하게 정렬 상태를 유지할 수 있습니다.
- 예외 케이스 가두기 : 현재 리스트의 그 어떤 값보다 큰 숫자가 들어왔을 때는 내부 if m <= a[i]: 조건문이 한 번도 참이 되지 못해 break를 타지 못합니다. 이 '만기 졸업' 상태를 낚아채어 맨 뒤에 .append(m) 처리를 해주는 예외 안전장치 설계가 필수적입니다.
```python
# [10-2] 정렬 없이 순서를 보존하는 전학생 삽입 메커니즘
a = [1, 4, 7, 10]
m = 12 # 최대값보다 큰 데이터 주입 상황
for i in range(len(a)):
    if m <= a[i]:
        a.insert(i, m); break
else:
    a.append(m) # break에 단 한 번도 안 걸려 만기 졸업했으므로 맨 뒤에 배치!
```

- 중복의 가차 없는 소멸 ([10-11]) : set(Iterable) 생성자를 통과하는 순간, 중복된 모든 주소 데이터는 내부 해시 테이블(Hash Table) 규칙에 의해 하나만 남고 증발하며 전체 길이(len)가 압축됩니다.
- 고속 연산자 기호 매칭 : 교집합은 &(.intersection()), 합집합은 |(.union()), 차집합은 (.difference()), 대칭차집합(서로 겹치지 않는 방들의 합)은 ^(.symmetric_difference()) 기호로 동작합니다.
- .add(x)의 유일성 보장 : 가변 객체인 세트에 원소를 실시간 추가할 때 사용하며, 중복된 값이 이미 슬롯에 존재한다면 에러 없이 조용히 무시(Ignore)합니다.
```python
# [10-12] 세트 교집합 연산 기반 중복 카운팅 기법
x = ("kim", "lee", "park", "song")
y = ("min", "park", "kong", "lew")
duplicate_count = len(set(x) & set(y)) # 결과: 1 (겹치는 'park'의 개수만 도출)
```

- 유실 데이터 추적 논리 ([10-13]) : range(1, N + 1)로 생성한 완벽한 마스터 데이터 세트에서, 입력받은 일부 리스트 컨테이너를 set(r)로 형변환하여 빼버리는 차집합() 연산을 취하면 유실되거나 낙오된 데이터를 O(N)의 고속 연산으로 정확하게 걸러낼 수 있습니다.
- sorted()의 비파괴 정렬 성질 : 세트 자료형은 순서 정보가 없는 불완전 컨테이너이므로 숫자를 순서대로 출력할 수 없습니다. 따라서 sorted(세트) 내장 함수를 통과시켜 원본 세트는 건드리지 않고, 정렬된 완벽한 새로운 리스트 객체를 반환받아 언패킹()으로 털어내는 것이 정석입니다.
```python
# [10-13] 유실물 추적용 세트 차집합 및 비파괴 정렬 조합
r = [1, 3, 4] # 2번이 빠진 리스트 상황
master_set = set(range(1, 5)) # {1, 2, 3, 4}
missing_items = sorted(master_set - set(r)) # 결과: [2] (리스트 형태로 정렬 반환)
```

- set(문자열)의 탐색 범위 압축 효과 ([10-15]) : 문자열 전체를 루프 돌려 .count()를 연타하면 중복된 글자를 검사할 때마다 텍스트 처음부터 끝까지 다시 읽는 최악의 O(N^2) 성능 저하가 일어납니다. 탐색 전 대상을 set(n)으로 묶어 중복 글자들을 전면 필터링해 주어야 고속 연산 장벽이 수립됩니다.
- 딕셔너리 생성 스타일 3가지 분류 :
```python
# [10-15] 딕셔너리 컴프리헨션을 활용한 압축 카운팅 종결
n = "AAABBC"
char_counts = {char: n.count(char) for char in set(n)} # 세트로 글자 종류를 압축 후 개수 매핑
# 출력 결과: {'A': 3, 'B': 2, 'C': 1} (세트의 무작위성 때문에 순서는 매번 달라질 수 있음)
```

💡 부족했던 디테일 한 스푼 보정 (pop()과 remove()의 안전장치 및 시간 복잡도)

- 리스트의 .remove()와 중간 방의 .pop(index)은 원소를 지운 뒤 메모리 공간을 메우기 위해 뒤쪽 데이터들을 한 칸씩 앞으로 땡기는 인덱스 리바인딩 연산을 수행합니다. 따라서 데이터가 커질수록 연산이 극도로 무거워지는 O(N)의 시간 복잡도를 가집니다. 반면, 인덱스가 없는 맨 끝방을 털어내는 .pop()은 주소록 재조정이 전혀 필요 없으므로 항상 O(1)의 초고속 속도를 보장합니다.
- 세트 연산 기호 중 차집합()을 쓸 때는 좌변과 우변이 모두 set 객체여야만 에러가 안 나지만, 내장 메서드인 s1.difference(r) 형태를 사용하면 인자 자리에 일반 리스트(r)나 튜플이 들어와도 파이썬이 내부적으로 자동 형변환하여 차집합을 계산해 주는 스펙적 유연함이 있습니다.
