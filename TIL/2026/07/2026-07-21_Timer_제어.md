## 📌 SysTick (System Tick) 타이머 요약 정리

### 1. SysTick이란?

- Cortex-M 코어 내장 타이머: STM32 주변장치(PERIPHERAL) 타이머와 달리 ARM Cortex-M CPU 코어 내부에 기본 탑재된 24비트 다운-카운터(Down-counter).
- 주요 용도:
### 2. 핵심 동작 방식 & 특징

- 24비트 감산(Down) 카운터: LOAD 값에서 시작하여 0을 향해 카운트다운.
- 자동 재적재 (Auto-reload): 0에 도달하면 LOAD 레지스터의 값이 VAL 레지스터로 자동 복사되어 다시 카운트다운 시작.
- N + 1 법칙: 0까지 세고 리셋되므로, N번 카운트하려면 LOAD 값에 N - 1을 설정해야 함.
### 3. 주요 4대 레지스터 (Register)

| 레지스터 | 비트 | 설명 |
|---|---|---|
| CTRL (Control & Status) | 32비트 | SysTick 동작 설정 및 상태 확인 (Enable, Clock Source, Interrupt, CountFlag) |
| LOAD (Reload Value) | 24비트 | 카운트 시작 초기값 (최대 0x00FFFFFF = 16,777,215) |
| VAL (Current Value) | 24비트 | 현재 카운트되고 있는 값 (Write 시 0으로 초기화 및 CountFlag Clear) |
| CALIB (Calibration) | 32비트 | 10ms 단위 틱 보정을 위한 팩토리 하드웨어 값 |

- Bit 0 (ENABLE): 1 = 타이머 동작 시작 / 0 = 정지
- Bit 1 (TICKINT): 1 = 0 도달 시 SysTick 예외(인터럽트) 발생 / 0 = 발생 안 함
- Bit 2 (CLKSOURCE): 1 = CPU 클럭(HCLK) 사용 / 0 = 외부 클럭(HCLK / 8) 사용
- Bit 16 (COUNTFLAG): 카운트가 0에 도달하면 1로 세팅됨 (레지스터를 Read하면 자동 0으로 Clear)
### 4. 시계 산출 공식 (HCLK / 8 기준)

\text{Reload Value (LOAD)} = \left( \frac{\text{HCLK}}{8} \div 1000 \right) \times \text{목표시간(ms)} - 1

- 예: HCLK = 168\text{MHz}인 경우
### 💬 한 줄 요약

> "SysTick은 ARM 코어 내부의 24비트 다운-카운터로, LOAD에서 0까지 세고 다시 자동으로 찰지게 되감아주는 정밀한 시계이다!"

```c
#include "device_driver.h"

void SysTick_Run(unsigned int msec)
{
	// Timer 설정 : 인터럽트 발생 안함, clock source는 HCLK/8, Timer 정지
	Macro_Write_Block(SysTick->CTRL, 0x7, 0x0, 0U);

	// 주어진 msec 값 만큼의 msec를 count하는 초기값 설정 (LOAD)
	SysTick->LOAD = (int)((HCLK / 8. / 1000.) + 0.5) * msec - 1;

	// VAL 레지스터 값 초기화(0) 및 COUNTFLAG Clear
	SysTick->VAL = 0;

	// Timer Start (시작이 되면 자동으로 LOAD의 값을 VAL로 가져간다)
	Macro_Set_Bit(SysTick->CTRL, 0U);
}

int SysTick_Check_Timeout(void)
{
	// Timer의 Timeout이 발생하면 참(1)리턴, 아니면 거짓(0) 리턴
	return Macro_Check_Bit_Set(SysTick->CTRL, 16U);
}

unsigned int SysTick_Get_Time(void)
{
	// Timer의 현재 count 값 리턴
	return SysTick->VAL;
}

unsigned int SysTick_Get_Load_Time(void)
{
	// Timer에 설정된 초기값을 리턴
	return SysTick->LOAD;
}

void SysTick_Stop(void)
{
	// Timer Stop
	Macro_Clear_Bit(SysTick->CTRL, 0U);
}
```

## 📌 STM32 TIMx (General-Purpose Timer) 요약 정리

### 1. TIMx 타이머란?

- STM32 주변장치(Peripheral) 타이머: ARM 코어 외부에 독립적으로 존재하는 범용 타이머.
- 주요 용도:
### 2. Time-Base Unit 3대 핵심 레지스터

| 레지스터 | 비트 | 이름 | 역할 |
|---|---|---|---|
| TIMx->CNT | 16/32비트 | Counter | 실시간으로 숫자를 세고 있는 '현재 시계 바늘' |
| TIMx->PSC | 16비트 | Prescaler | 원본 클럭을 쪼개어 타이머 카운트 속도를 조절하는 '분주기' |
| TIMx->ARR | 16/32비트 | Auto-Reload Register | 카운트의 목표 지점 / 리셋 기준선 (초기 적재 값) |

### 3. 카운트 동작 방식 3가지

- 0 \longrightarrow \text{ARR} 방향으로 증가
- CNT가 ARR 값에 도달하면 0으로 리셋되며 오버플로우(Overflow) / UEV(Update Event) 발생
- \text{ARR} \longrightarrow 0 방향으로 감소
- CNT가 0에 도달하면 ARR 값으로 리셋되며 언더플로우(Underflow) / UEV(Update Event) 발생
- 0 \longrightarrow \text{ARR} \longrightarrow 0 (UP과 DOWN을 반복)
- 정밀한 PWM 주파수 제어 및 모터 드라이브 제어에 사용
### 4. 타이머 주기(시간) 계산 공식

타이머의 최종 주기 T (초 단위) 및 주파수 f 계산식:

T = \frac{(\text{PSC} + 1) \times (\text{ARR} + 1)}{f_{\text{TIM\_CLK}}}

- f_{\text{TIM\_CLK}}: 타이머로 들어오는 공급 클럭 주파수 (예: 84MHz)
- +1을 하는 이유: 레지스터 설정값 0이 실제로는 '1번의 클럭 펄스'를 의미하기 때문 (N을 세려면 레지스터에 N-1 저장)
1. PSC = 8399 세팅 \rightarrow 84\,\text{MHz} \div (8399 + 1) = 10\,\text{kHz} (0.1\,\text{ms} 마다 1카운트)
1. ARR = 9999 세팅 \rightarrow 10\,\text{kHz} \div (9999 + 1) = 1\,\text{Hz} (1\,\text{초} 완료)
### 5. 주요 이벤트 및 플래그 (Flag)

- UEV (Update Event): 카운터가 ARR 도달(UP) 또는 0 도달(DOWN) 시 발생하는 하드웨어 이벤트.
- UIF (Update Interrupt Flag): TIMx->SR (Status Register)의 Bit 0. UEV가 발생하면 1로 세팅되며, 소프트웨어에서 처리 후 0으로 직접 Clear 해주어야 함.

### 6. 주요 제어/이벤트 레지스터 (추가)

| 레지스터 | 비트 | 이름 | 역할 |
|---|---|---|---|
| TIMx->CR1 | 16비트 | Control Register 1 | 타이머의 동작 방식을 총괄 설정하는 메인 컨트롤러 (ON/OFF, 카운트 방향, 정지 조건 등) |
| TIMx->EGR | 16비트 | Event Generation Register | 소프트웨어로 강제로 이벤트를 찔러 넣어주는 이벤트 발생기 |

### 🔍 TIMx->CR1 (Control Register 1) 주요 비트

타이머 동작 시 가장 자주 건드리는 핵심 비트들입니다.

- Bit 0 (CEN - Counter Enable): 타이머 ON/OFF 스위치
- Bit 4 (DIR - Direction): 카운트 방향 설정
- Bit 3 (OPM - One-pulse Mode): 단발성 동작 설정
- Bit 7 (ARPE - Auto-reload Preload Enable): ARR 버퍼링 설정
### 🔍 TIMx->EGR (Event Generation Register) 주요 비트

소프트웨어적으로 타이머의 상태를 즉시 리셋하거나 강제로 업데이트할 때 사용합니다. (Write-Only 레지스터)

- Bit 0 (UG - Update Generation): 강제 업데이트 이벤트 발생
### 💡 Hz(헤르츠) 직관적 이해하기

- 1\text{Hz} = 1초에 펄스가 1번 뜀 (1\text{초}에 카운트 1개)
- 10\text{Hz} = 1초에 펄스가 10번 뜀 (0.1\text{초}마다 카운트 1개)
- 1\,\text{kHz} (1,000\text{Hz}) = 1초에 펄스가 1,000번 뜀 (1\text{ms}마다 카운트 1개)
- 50\,\text{kHz} (50,000\text{Hz}) = 1초에 펄스가 50,000번 뜀 (20,\mu\text{s}마다 카운트 1개) 👈 방금 구현하신 설정!
### 🔄 주파수(Hz)와 시간(주기, T)의 관계

주파수(f)와 펄스 1개가 뛰는 데 걸리는 시간인 주기(T)는 서로 역수 관계입니다.

f = \frac{1}{T} \quad \Longleftrightarrow \quad T = \frac{1}{f}

1. "주파수(50,000\text{Hz})로 1\text{ms} 동안 뛰는 펄스 수 구하기"
