# 📡 UART vs RS-232 통신 완벽 정리

## 📌 1. 직관적 표

| 구분 | UART (Universal Asynchronous Receiver-Transmitter) |
|---|---|
| 정의 | MCU 내부에 존재하는 디지털 통신 모듈/프로토콜 |
| 신호 형태 | 디지털 로직 신호 (TTL 레벨) |
| 전압 범위 | 0V (Low / Space)<br><br>3.3V 또는 5V (High / Mark) |
| 전송 거리 | 극도로 짧음 (보드 내부, 몇 cm ~ 몇십 cm) |
| 노이즈 내성 | 매우 약함 (전압차가 작아 외부 노이즈에 취약) |
| 주요 용도 | MCU ↔ 센서, MCU ↔ 블루투스/Wi-Fi 모듈 |

## 📌 2. 핵심 개념 정리

### UART (비동기 리시버/트랜스미터)

- 동기화 시계(Clock) 선이 없음: TxD(송신)와 RxD(수신) 2개의 라인으로만 통신합니다.
- 보레이트(Baud Rate) 필수: 클럭 선이 없기 때문에, 송신측과 수신측이 "1초에 몇 비트를 보낼지(예: 115200bps)" 미리 똑같이 맞춰두어야 데이터가 깨지지 않습니다.
- TTL(Transistor-Transistor Logic) 레벨: MCU가 다루는 전압인 0V와 3.3V(또나 5V)를 그대로 출력합니다.
## 📌 4. UART 데이터 프레임 구조 (Data Frame)

UART 통신 시 1바이트(8비트)의 데이터가 선을 타 이동할 때의 시퀀스입니다.

```plain text
Idle (High) ──┐        ┌───┬───┬───┬───┬───┬───┬───┬───┐       ┌── Idle (High)
              │ Start  │ D0│ D1│ D2│ D3│ D4│ D5│ D6│ D7│ Stop  │
              └────────┴───┴───┴───┴───┴───┴───┴───┘       └───
               (1 Bit)        Data Bits (5~8 Bits)   (1~2 Bits)
```

1. Idle State (대기 상태): 통신을 하지 않을 때 선은 항상 High(1) 상태를 유지합니다.
1. Start Bit (시작 비트): 송신측에서 선을 Low(0)로 1비트 시간 동안 떨어뜨려 "데이터 시작한다!"고 수신측에 알립니다.
1. Data Bits (데이터 비트): LSB(최하위 비트)부터 MSB(최상위 비트) 순서대로 보통 8비트 데이터 전송.
1. Parity Bit (패리티 비트 - 선택): 데이터 오류 검증용 (Even / Odd / None).
1. Stop Bit (정지 비트): 데이터 전송이 끝났음을 알리기 위해 선을 다시 High(1)로 1~2비트 동안 올려 유지합니다.
## ✍️ 요약

> • UART: MCU 내부에서 **디지털 데이터(0V/3.3V)**를 직렬로 뿜어내는 통신 방식.
• RS-232: UART의 신호를 멀리 보내기 위해 **전압(-12V/+12V)**을 튀겨서 노이즈에 강하게 만든 물리적 규격.
• 핵심 포인트: 둘을 연결할 때는 전압 차이로 인해 MCU가 타버릴 수 있으므로 중간에 MAX232 같은 레벨 변환 IC가 필수적이다.

### 📡 UART vs USART (3초 요약)

- UART (비동기): 클럭(시계) 선이 없음.
- USART (동기/비동기 겸용): 클럭(시계) 선이 있음.
### 💡 한 줄 결론

> USART는 UART의 업그레이드 버전!

```c
#include "device_driver.h"
#include "Option.h" // PCLK2 등 클럭 상수가 정의된 헤더

void Uart1_Init(int baud)
{
    /* 1. Baud Rate 계산에 필요한 변수 선언 */
    double div;         // 전체 분주비 실수값
    unsigned int mant;  // 분주비 정수부 (Mantissa)
    unsigned int frac;  // 분주비 소수부 (Fraction)

    /* 2. 클럭(전원) 활성화 */
    Macro_Set_Bit(RCC->AHB1ENR, 0);  // GPIOA 포트 클럭 ON (Bit 0)
    Macro_Set_Bit(RCC->APB2ENR, 4);  // USART1 모듈 클럭 ON (Bit 4)

    /* 3. GPIO 핀 설정 (PA9 = TX, PA10 = RX) */
    Macro_Write_Block(GPIOA->MODER, 0xf, 0xa, 18);   // PA9, PA10를 Alternate Function 모드(10)로 설정
    Macro_Write_Block(GPIOA->AFR[1], 0xff, 0x77, 4); // PA9, PA10에 AF7 (USART1 기능) 연결
    Macro_Write_Block(GPIOA->PUPDR, 0xf, 0x5, 18);   // PA9, PA10에 내부 Pull-Up 저항 설정

    /* 4. Baud Rate (통신 속도) 계산 및 레지스터 설정 */
    div = PCLK2 / (16. * baud);                      // 실수 형태의 전체 분주비 계산 (96MHz / (16 * baud))
    mant = (int)div;                                 // 정수부만 싹 잘라서 extraction
    frac = (int)((div - mant) * 16 + 0.5);           // 소수부에 16을 곱하고 반올림(+0.5) 연산
    mant += frac >> 4;                               // 소수부 반올림으로 인한 올림값 처리 (16 이상 시 정수부 +1)
    frac &= 0xf;                                     // 소수부를 4비트(0~15) 크기 마스크 처리
    
    USART1->BRR = (mant << 4) | (frac << 0);         // BRR 레지스터에 정수부와 소수부 값을 대입 (115200 기준 0x341)

    /* 5. USART 제어 레지스터 설정 (8N1 설정 및 모듈 활성화) */
    USART1->CR1 = (1 << 13) | (0 << 12) | (0 << 10) | (1 << 3) | (1 << 2); 
    // Bit 13 (UE=1): USART1 모듈 켜기
    // Bit 12 (M=0) : Data Bit 8비트 설정
    // Bit 10 (PCE=0): Parity Bit 안 씀 (No Parity)
    // Bit 3  (TE=1): 송신(TX) 기능 활성화
    // Bit 2  (RE=1): 수신(RX) 기능 활성화

    USART1->CR2 = 0 << 12; // Bit 12~13 (STOP=00): 1 Stop Bit 설정
    USART1->CR3 = 0;       // 하드웨어 흐름제어(RTS/CTS) 및 DMA 비활성화 (기본 통신 모드)
}
```
