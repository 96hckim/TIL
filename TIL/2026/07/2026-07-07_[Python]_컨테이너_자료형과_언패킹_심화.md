### [3-1] 4대 컨테이너 자료형의 생성 및 특징

- 파이썬은 데이터를 목적에 따라 묶을 수 있는 4가지 핵심 컨테이너를 제공합니다.
- 중복 제거 확인: Set(s)은 중복된 값(20)을 허용하지 않으므로 출력 시 자동으로 하나가 제거됩니다.
- 내장 함수 활용: len()은 데이터 개수, max()/min()은 최댓값/최솟값, sum()은 합계를 구해줍니다. (단, 딕셔너리는 Key-Value 쌍이므로 sum 연산의 대상이 다릅니다.)
| 자료형 | 기호 | 순서(Index) | 중복 허용 | 수정 가능 여부 (Mutable) |
|---|---|---|---|---|
| Tuple (튜플) | ( ) | 있음 | 허용 | 불가능 (Immutable) |
| List (리스트) | [ ] | 있음 | 허용 | 가능 (Mutable) |
| Set (집합) | { } | 없음 | 불허 (중복 제거) | 가능 (Mutable) |
| Dictionary (딕셔너리) | {K:V} | 없음 (Key 기반) | Key 중복 불가 | 가능 (Mutable) |

```python
# 1. 네 가지 자료형의 기본 형태 선언
t = (10, 20, 30, 20)
l = [10, 20, 30, 20]
s = {10, 20, 30, 20] # ⚠️ Set은 중복을 허용하지 않으므로 {10, 20, 30}만 저장됨
d = {1: 2, "a": 10, "Hello": "Kim"}

# 2. 내장 함수를 이용한 특징 분석 출력
print(type(t), t, len(t), max(t), sum(t))
print(type(l), l, len(l), min(l), sum(t))
print(type(s), s, len(s), min(s), sum(s)) # 개수(len)가 3개로 출력되는 것 확인!
print(type(d), d, len(d))                  # 딕셔너리의 len은 Key-Value 쌍의 개수
```

### 💡 [주의] 빈 컨테이너 및 1개짜리 컨테이너 선언 시 특이 사항

- 빈 집합 생성 규칙: 빈 딕셔너리가 {} 기호를 선점했기 때문에, 빈 집합을 만들 때는 반드시 기호 대신 set() 함수를 사용해야 합니다.
- 1개짜리 튜플 생성 규칙: 튜플에 요소를 딱 하나만 넣을 때는 단순 괄호 연산자((10))와 구분하기 위해 반드시 숫자 뒤에 쉼표 ,를 붙여야 튜플로 인식됩니다.
```python
# 1. 빈 컨테이너 만들기
t, l, s, d = (), [], set(), {} # ⚠️ set() 명시 중요! {}만 쓰면 딕셔너리가 됩니다.
print(t, l, s, d)

# 2. 요소가 딱 1개인 컨테이너 만들기
t, l, s, d = (10,), [1], {2}, {1: 2} # ⚠️ 튜플은 '(10,)' 처럼 콤마를 붙여야 튜플 객체가 됩니다.
print(t, l, s, d)
```

### [3-2] 컨테이너의 아이템 접근 (Indexing)

- 시퀀스 자료형 (문자열, 튜플, 리스트): 순서(Index)가 있으므로 대괄호 [ ] 안에 방 번호를 넣어 접근합니다. 음수 인덱스(1)를 사용하면 맨 뒤에서부터 역순으로 접근할 수 있습니다.
- 딕셔너리: 순서 대신 고유한 Key 값을 대괄호 [ ] 안에 넣어서 매칭된 Value를 찾습니다.
- 집합 (Set): 내부적으로 순서가 존재하지 않는 해시 테이블이기 때문에 s[0] 같은 인덱싱 접근이 아예 불가능합니다.
```python
c = "Hello"
t = (1, 3.14, "name")
lst = [1, 3.14, "name"] # 💡 l 변수명을 lst로 변경
k = {"id": 1, 2: 3.14, "tag": "name"}
s = {1, 3.14, "name", 1, 3.14}

print(c[0], c[1], c[-1], c[-2]) # 문자열 인덱싱 (H e o l)
print(t[0], t[1], t[-1], t[-2]) # 튜플 인덱싱 (1 3.14 name 3.14)
print(lst[0], lst[1], lst[-1], lst[-2]) # 리스트 인덱싱 (1 3.14 name 3.14)
print(k["id"], k[2], k["tag"])  # 딕셔너리 Key 조회 (1 3.14 name)
print(s)                        # 집합은 순서 없이 중복만 제거되어 출력됨
```

### [3-3] 가변(Mutable) vs 불변(Immutable) 데이터 수정

- 불변 객체 (문자열, 튜플): 태생적으로 메모리에 한 번 올라가면 내부 값을 절대 바꿀 수 없습니다. 수정하려고 시도하면 TypeError가 발생합니다.
- 가변 객체 (리스트, 딕셔너리, 집합): 언제든지 방 안의 알맹이를 다른 데이터로 덮어쓰거나 수정할 수 있습니다.
```python
c = "Hello"
t = (1, 3.14, "name")
lst = [1, 3.14, "name"]
k = {"id": 1, 1: 3.14, "tag": "name"}

# c[0] = "K"     # ❌ TypeError! 문자열은 원본 수정 불가
# t[1] = 5.125   # ❌ TypeError! 튜플은 원본 수정 불가

lst[2] = "score" # ✨ 리스트 원본 수정 성공!
k["id"] = 100   # ✨ 딕셔너리 기존 Key의 값 변경 성공!

print(lst, k, sep="\n")
```

### [3-4] 가변 컨테이너의 아이템 추가 (append, insert, add)

- lst.append(값): 리스트의 맨 뒤에 새로운 값을 붙입니다. (O(1) 속도)
- lst.insert(인덱스, 값): 리스트의 원하는 중간 위치에 값을 비집고 집어넣습니다. 뒤의 데이터가 한 칸씩 밀리므로 데이터가 많을 땐 느려집니다. (O(n) 속도)
- k[새로운Key] = 값: 딕셔너리에 기존에 없던 Key와 밸류를 쌍으로 추가합니다.
- s.add(값): 집합(Set)에 새로운 원소를 추가합니다. (이미 있는 값이면 중복이라 무시됨)
```python
lst = [1, 3.14, "name"]
k = {"id": 1, "rate": 3.14, "tag": "name"}
s = {1, 3, 5, 2}

lst.append("song")   # 맨 뒤에 추가 -> [1, 3.14, 'name', 'song']
lst.insert(1, "lew") # 1번 인덱스 자리에 새치기 -> [1, 'lew', 3.14, 'name', 'song']

k["num"] = 200       # 새로운 Key-Value 등록
s.add(10)            # 집합에 원소 추가

print(lst, k, s, sep="\n")
```

### [3-5] 실습: 리스트 줄 세우기 (새치기 알고리즘)

- 사용자에게 [등수(번호) 이름]을 입력받아 대기 명단 리스트의 중간에 끼워 넣는 현실적인 예제입니다.
- 컴퓨터와 사람의 계산 차이 보정: 사람은 첫 번째 등수를 1등이라고 부르지만 컴퓨터의 인덱스는 0부터 시작합니다. 따라서 사용자가 3등(3)을 입력하면 컴퓨터 내부적으로는 3 - 1 = 2번 인덱스 자리에 끼워 넣어야 사람이 생각하는 정확한 자리에 들어갑니다.
```python
# 기존 대기자 명단 (0번: kim, 1번: lee, 2번: moon ...)
lst = ["kim", "lee", "moon", "lew", "kang", "song"]

# 1. 한 줄에 등수와 이름을 공백으로 분리해서 입력받음 (예: 3 park)
num, name = input().split()
num = int(num) # 등수는 숫자로 계산해야 하므로 정수 변환

# 2. 사용자가 말한 등수에서 1을 빼서 컴퓨터 방 번호(인덱스)를 맞춘 뒤 insert
lst.insert(num - 1, name)

print(lst)
# 입력 예시: 3 park
# 출력 결과: ['kim', 'lee', 'park', 'moon', 'lew', 'kang', 'song'] -> 원래 3등이던 moon 앞에 정확히 안착!
```

## [3-6] 다중 컨테이너 생성과 아이템 접근 (다차원 인덱싱)

- 리스트 안에 리스트가 들어가거나 튜플 안에 딕셔너리가 들어가는 등, 컨테이너 내부에 또 다른 컨테이너를 자유롭게 배치할 수 있습니다.
- 다차원 접근: 알맹이에 접근할 때는 대괄호를 연속으로 붙여서 [바깥방_번호][안쪽방_번호] 구조로 찾아갑니다.
- 문자열 인덱싱: 안쪽 데이터가 문자열("name")일 때, 한 번 더 대괄호를 붙이면 문자열의 알파벳 글자 하나("name"[3] -> 'e')까지 쏙 뽑아낼 수 있습니다.
```python
t1 = ((1, 3.14, "name"), (10,), ("kim", 30, 5.12))
t2 = ([1, 3.14, "name"], "Yoon", {"a": 1, "b": 3.14})
l1 = [[1, 3.14, "name"], [10], ["kim", 30, 5.12]]
l2 = [(1, 3.14, "name"), "Yoon", {"a": 1, "b": 3.14}]

# 1. 2차원 튜플/리스트 기본 출력
print(t1, t2, sep="\n")

# 2. 연속 대괄호를 이용한 심층 데이터 접근
print(t1[0][0], t1[-1][-1])    # 1과 5.12 출력
print(t2[0][2][3], t2[2]["b"])  # "name"의 3번 글자인 'e'와 딕셔너리 Key "b"의 값인 3.14 출력

print(l1, l2, sep="\n")
print(l1[0][0], l1[-1][-1])    # 1과 5.12 출력
print(l2[0][2][3], l2[2]["b"])  # 'e'와 3.14 출력
```

### [3-7] 튜플의 패킹(Packing)과 언패킹(Unpacking)

- 패킹 (Packing): 여러 개의 값을 괄호 ( ) 없이 쉼표 ,로만 나열해도 파이썬이 알아서 하나의 튜플 바구니로 꽁꽁 싸맵니다.
- 언패킹 (Unpacking): 튜플 바구니에 든 알맹이들을 여러 변수에 동시에 흩뿌려 담아줄 수 있습니다. 이때 왼쪽의 변수 개수와 오른쪽 데이터의 개수가 정확히 일치해야 합니다.
- 문자열 분해: 일반 문자열("kim")도 길이가 3짜리 시퀀스 데이터이므로 글자 수에 맞춰 3개의 변수를 준비하면 철자가 하나씩 쪼개져 들어가는 언패킹이 일어납니다.
```python
# 1. 튜플 패킹 (괄호를 안 써도 자동으로 튜플로 묶임)
t1 = (1, 2, 3)
t2 = 1, 2, 3
t3 = "kim", "lee", "lew"
print(type(t1), t1)
print(type(t2), t2)
print(type(t3), t3)

# 2. 튜플 언패킹 (바구니를 해체하여 각 변수에 나눠 담기)
t4 = 1, 2, 3
a, b, c = t4
print(a, b, c) # 1 2 3

# 3. 문자열 언패킹 (글자가 하나씩 쪼개져 들어감)
a, b, c = "kim"
print(a, b, c) # k i m
```

### [3-8] 파이썬식 두 변수의 값 바꾸기 (Swap)

- C언어에서는 두 변수의 값을 바꾸려면 임시 저장 상자인 temp 변수를 만들어서 3줄에 걸쳐 값을 돌려 막아야 했습니다. (temp = a; a = b; b = temp;)
- 파이썬은 위에서 배운 [오른쪽 패킹 ➡️ 왼쪽 언패킹] 메커니즘을 동시작업(원자적 연산)으로 처리하기 때문에, 임시 변수 없이 단 한 줄 a, b = b, a 만으로 두 변수의 값을 안전하게 뒤바꿀 수 있습니다.
```python
a = 10
b = 20
print(a, b) # 10 20

# 파이썬 고유의 문법으로 순식간에 값 체인지 (오른쪽에서 20, 10 튜플이 패킹되고 왼쪽에 언패킹됨)
a, b = b, a

print(a, b) # 20 10
```

### [3-9] 다양한 Iterable(반복 가능) 타입의 언패킹

- 튜플이나 리스트뿐만 아니라 Set, Dictionary, 문자열 등 내부에 데이터를 순회할 수 있는 모든 객체(Iterable)는 구조 분해(언패킹)를 지원합니다.
- 딕셔너리 언패킹의 특이 사항: 딕셔너리를 그냥 언패킹하면 Value(값)는 버려지고 Key(키)만 쪼개져서 변수에 들어갑니다.
- Set 언패킹의 주의 점: 집합(Set)은 순서가 없기 때문에 언패킹했을 때 어떤 변수에 어떤 값이 매칭될지 보장할 수 없습니다.
```python
t1, t2, t3 = (1, 2, 3)
l1, l2, l3 = [1, 2, 3]
s1, s2, s3 = {1, 2, 3}
d1, d2, d3 = {"a": 1, "b": 2, "c": 3} # 💡 딕셔너리는 'Key'만 언패킹됨!
c1, c2, c3 = "str"

print(t1, t2, t3, l1, l2, l3, s1, s2, s3, d1, d2, d3, c1, c2, c3)
# 출력 결과 중 d1, d2, d3 부분은 'a', 'b', 'c' 가 나옵니다.
```

### [3-10] 딕셔너리 전용 View 메서드 (keys, values, items)

- 딕셔너리에서 Key만, Value만, 혹은 둘 다 세트로 뽑아내기 위해 3가지 전용 메서드를 제공하며, 이를 딕셔너리 뷰(Dict View)라고 부릅니다.
- d.keys(): 키 목록만 반환
- d.values(): 값 목록만 반환
- d.items(): 키와 값이 튜플로 묶인 (Key, Value) 쌍 목록을 반환
- 애스터리스크* 패킹 풀기: 프린트문 안에 k 형태로 작성하면 뷰 객체의 껍데기를 한 꺼풀 벗겨서 알맹이만 공백으로 구분해 출력해 주는 파이썬 특수 문법입니다.
```python
d = {"a": 1, "b": 2}

k = d.keys()
v = d.values()
i = d.items()

# 1. 뷰 객체의 타입과 알맹이 풀어서(*) 출력 확인
print(type(k), k, *k) # <class 'dict_keys'> dict_keys(['a', 'b']) a b
print(type(v), v, *v) # <class 'dict_values'> dict_values([1, 2]) 1 2
print(type(i), i, *i) # <class 'dict_items'> dict_items([('a', 1), ('b', 2)]) ('a', 1) ('b', 2)

# 2. 각 뷰 메서드의 결과를 개별 변수에 언패킹
a1, a2 = d.keys()   # 'a', 'b'
b1, b2 = d.values() # 1, 2
c1, c2 = d.items()  # ('a', 1), ('b', 2) -> 각 변수에 튜플로 들어감

print(a1, a2, b1, b2, c1, c2)
```

### [3-11] 나머지 싹 다 모아 담기: 부분 언패킹 *

- 언패킹할 변수 앞에 별표*를 하나 붙여두면, "앞뒤 변수들이 고정으로 가져간 데이터를 제외한 나머지 모든 찌꺼기 데이터를 몽땅 리스트로 묶어서 내가 다 흡수하겠다"는 의미가 됩니다.
- 이 기법을 사용하면 개수가 유동적인 데이터 무리에서 특정 위치의 고정 데이터만 칼같이 도려낼 수 있습니다.
```python
t = (10, 20, 30, 40, 50)

a, *b = t
print(a, b) # a=10 / b=[20, 30, 40, 50] (맨 앞 빼고 다 가져옴)

*a, b = t
print(a, b) # a=[10, 20, 30, 40] / b=50 (맨 뒤 빼고 다 가져옴)

a, b, *c, d, e = t
print(a, b, c, d, e) # a=10, b=20 / c=[30] / d=40, e=50

a, b, c, *d, e, f = t
print(a, b, c, d, e, f) # a=10, b=20, c=30 / d=[] (담을 게 없으면 빈 리스트가 됨) / e=40, f=50
```

### [3-12] 실습: 일등과 꼴찌만 기억하는 사회 (알고리즘 구현)

- 사용자가 한 줄에 여러 개의 점수나 명단을 공백으로 구분해 무작위로 입력했을 때, 중간에 낀 데이터 개수가 몇 개이든 상관없이 맨 앞(일등)과 맨 뒤(꼴찌)만 정확하게 변수에 골라 담아 출력하는 세련된 한 줄 구현 코드입니다.
```python
# 사용자가 명단을 공백으로 분리하여 입력 (예: 홍길동 김철수 이영희 박민수 최성재)
first, *x, last = input().split()

# 맨 처음 단어는 first에, 맨 마지막 단어는 last에 들어가고 중간 모든 명단은 x 리스트에 격리됨
print(first, last)
```

### [3-13] 컨테이너 변환 함수의 동작 이해 (str, tuple, list)

- str() 변환의 함정: 튜플이나 리스트를 str() 함수로 감싸면 데이터가 분해되는 것이 아니라, 괄호와 쉼표를 포함한 글자 껍데기 전체가 통째로 하나의 문자열("['1', '2']")로 변해버립니다. (텍스트 파일 저장 시 주로 사용)
- tuple(), list() 변환: 리스트를 튜플로 바꾸거나 튜플을 리스트로 바꿀 때는 차원(깊이) 구조를 그대로 유지한 채 가장 바깥쪽의 바구니 형태만 쏙 교체됩니다.
```python
t1 = (1, 2, 3, 4)
t2 = ((1, 2, 3), (4, 8), (1, 2, 3), (6,))
l1 = [1, 2, 3, 4, 4, 4, 5]
l2 = [[1, 2, 3], [4, 8], [6], [4, 8]]

# 1. str() 함수 사용: 컨테이너 모양 그대로 "글자 덩어리"가 됨
a, b, c = str(3.14), str(t1), str(t2)
d, e = str(l1), str(l2)
print(a, b, c, d, e, sep="\n")

# 2. tuple() 및 list() 함수 사용: 바깥 바구니 종류만 교체
t3, t4 = tuple(l1), tuple(l2) # 리스트가 튜플로 변환됨
lst3, lst4 = list(l1), list(l2) # 튜플이 리스트로 변환됨 (l3, l4 변수명 수정)
print(t3, t4, lst3, lst4, sep="\n")
```

### [3-14] 실습: 튜플 명단에 새치기 구현하기 (Casting 기법)

- 앞서 공부했듯이 튜플(tuple)은 불변(Immutable) 객체이므로 insert()나 append() 같은 수정 메서드를 원천적으로 지원하지 않습니다.
- 우회 해결책 (Casting): 수정이 불가능한 튜플을 잠시 수정 가능한 리스트(list)로 변환하여 새치기(insert) 연산을 수행한 뒤, 다시 원래대로 튜플(tuple)로 포장해 주는 테크닉입니다. 데이터의 무결성을 지키면서 변형을 가할 때 실무에서 정말 자주 쓰는 패턴입니다.
```python
# 수정이 불가능한 원본 튜플 대기 명단
t = ("kim", "lee", "moon", "lew", "kang", "song")

# 1. 한 줄에 등수와 이름을 공백으로 분리해서 입력받음 (예: 3 park)
num, name = input().split()
num = int(num)

# 2. [핵심] 튜플을 가변 객체인 리스트로 변환합니다.
t_list = list(t)

# 3. 리스트 상태에서 인덱스 보정(num - 1) 후 새치기 삽입을 진행합니다.
t_list.insert(num - 1, name)

# 4. 작업이 끝난 리스트를 다시 안전한 불변 객체인 튜플로 원상복구합니다.
t = tuple(t_list)

# 5. 최종 결과 출력
print(t)
```

### [3-15] 딕셔너리(dict) 변환 함수의 조건과 특징

- 2차원 쌍 구조의 필수성: 리스트나 튜플을 딕셔너리로 바꿀 때는 내부 데이터가 반드시 [Key, Value] 또는 (Key, Value) 형태의 2글자 쌍(Pair)으로 묶여 있어야 합니다.
- 중복 Key 덮어쓰기: 변환하려는 데이터에 동일한 Key가 여러 개 존재하면 에러가 나는 것이 아니라, 가장 마지막에 등장한 값으로 덮어써집니다.
- 키워드 인자(kwargs) 변환: dict(a=10, b=20) 형식으로 함수 호출하듯이 선언하면 변수명이 자동으로 문자열 Key가 되는 편리한 딕셔너리 생성 기법을 지원합니다.
```python
l1 = [["a", 1], ["b", 2], ["c", 3]]
t1 = (("a", 10), (1, 5), (3.14, "A"))

# 1. 정상적인 2차원 쌍 구조 변환
x = dict(l1), dict(t1)
print(*x, sep="\n")

# 2. 중복된 Key가 존재할 때 (가장 뒤에 있는 값인 40과 5가 최종 저장됨)
l2 = [["a", 10], ("c", 20), ["c", 40]]
t2 = (["a", 10], ("a", 5), ("c", 40))

x = dict(l2), dict(t2)
print(*x, sep="\n")

# 3. 안쪽 데이터의 타입이 달라도 쌍(2개)만 맞으면 변환 가능
t3 = (((1, 2), "abc"), ((3, 4, 5), 10))
print(dict(t3)) # {(1, 2): 'abc', (3, 4, 5): 10}

# 4. 키워드 인자 방식으로 직관적인 생성
d = dict(a=10, b=3.5, xy="kim")
print(d) # {'a': 10, 'b': 3.5, 'xy': 'kim'}
```

### [3-16, 17] if문 없이 매핑하기: 룩업 테이블(Lookup Table) 알고리즘

- Booleans는 사실 숫자다: 파이썬 내부적으로 True는 숫자 1, False는 숫자 0과 완벽히 호환됩니다. 이를 이용해 t[n >= 0] 처럼 대괄호 안에 조건식을 넣으면 인덱스 0 또는 1번이 작동합니다.
- 룩업 테이블 기법: 데이터와 결과값을 미리 딕셔너리나 튜플로 저장해 둔 뒤, 입력값(Key)을 통해 즉시 결과(Value)를 찾아내는 방식입니다. if s == "apple": print(10) 같은 복잡한 조건문 연산을 제거하여 코드 속도와 가독성을 획기적으로 올려줍니다.
```python
# [3-16] 불리언(True/False) 인덱싱을 활용한 룩업
n = 10
t = ("LOW", "HIGH")
print(t[n >= 0]) # 10 >= 0 은 True(1) 이므로 t[1] 인 'HIGH' 출력

d = {False: "LOW", True: "HIGH"}
print(d[n >= 0]) # d[True] 의 값인 'HIGH' 출력

# [3-17] 실습: 과일 코드 룩업 테이블 검색
s = input() # 예: apple 입력

table = {"apple": 10, "mango": 12, "orange": 17}
print(table[s]) # 입력된 문자를 Key로 사용해 조건문 없이 10 출력
```

### [3-18, 19] 메모리를 아끼는 마법: range 함수의 메커니즘

- Lazy Evaluation (지연 평가): range(100000000)은 1억 개의 숫자를 메모리에 진짜로 만드는 것이 아닙니다. 단지 "시작, 끝, 증감 폭" 정보만 가지고 있다가 루프를 돌 때만 숫자를 하나씩 실시간으로 생성합니다.
- 고정된 메모리 크기: 숫자의 범위가 100개이든 1억 개이든 상관없이 range 객체가 차지하는 메모리 크기(sys.getsizeof)는 48바이트(또는 환경에 따라 고정된 크기)로 완전히 동일합니다.
- 시퀀스 속성 지원: 실제 리스트는 아니지만 인덱싱(a[1]), 언패킹(d, *e, f = a), 슬라이싱이 모두 가능합니다. 알맹이를 눈으로 진짜 확인하고 싶을 때는 애스터리스크*를 붙여서 언패킹 출력을 해야 합니다.
```python
import sys

# [3-18] 숫자의 범위와 관계없이 일정한 메모리 공간만 사용하는 것 확인
print(range(5))
print(sys.getsizeof(range(100)))          # 범위가 작을 때의 메모리 크기
print(sys.getsizeof(range(100000000)))    # 범위가 1억 배 커져도 메모리 크기는 고정!

# [3-19] range 객체의 인덱싱 및 다양한 활용 패턴
a = range(1, 5) # 1, 2, 3, 4
b, c = range(2) # 0, 1 언패킹
d, *e, f = a    # d=1, e=[2, 3], f=4 부분 언패킹

print(a[1], b, c) # range도 인덱스 접근 가능 (2 0 1 출력)
print(d, e, f)
print(a)          # 그냥 출력하면 모양새(range(1, 5))만 나옴
print(*a)         # 앞에 *를 붙여 언패킹해야 1 2 3 4 내용물이 출력됨

# 다양한 range 조건 생성 패턴 (* 기호로 압축 해제해서 출력)
print(*range(5))        # 0부터 시작: 0 1 2 3 4
print(*range(2, 5))     # 시작, 끝 지정: 2 3 4 (끝 번호 미포함)
print(*range(0, 6, 2))  # 증감 폭(step) 지정: 0 2 4
print(*range(4, -1, -1)) # 역순 출력 패턴: 4 3 2 1 0
```

# 📦 Chapter 3. 컨테이너 핵심 요약

## 1. 4대 컨테이너 특징

- list [ ] : 순서 있음, 중복 허용, Mutable (수정 가능)
- tuple ( ) : 순서 있음, 중복 허용, Immutable (수정 불가) ➡️ (10,) 1개짜리 선언 시 쉼표 필수.
- set { } : 순서 없음, 중복 제거, 빈 집합은 반드시 set() 선언.
- dict {K:V} : 순서 없음(Key 기반), Key 중복 불가(마지막 값으로 덮어씀).
## 2. 아이템 제어 & 연산

- 조회 : 시퀀스형은 인덱싱(lst[0], lst[-1]), 딕셔너리는 키 조회(d["Key"]), 집합은 인덱싱 불가.
- 추가 : lst.append() (맨 뒤, O(1)) / lst.insert() (중간 새치기, O(n)) / set.add() / dict[NewKey] = Value.
- 우회 수정(Casting) : 불변 객체인 튜플을 수정할 때는 list(t) 변환 ➡️ insert() ➡️ tuple(t) 원상복구 프로세스 활용.
## 3. 패킹 & 언패킹 (Structure)

- 패킹 : 괄호 없이 ,만 나열해도 자동으로 튜플로 묶임.
- SWAP : a, b = b, a (임시 변수 없이 한 줄로 값 교환).
