## 1. 수학 및 연산 관련 내장 함수

### [5-1] 수학적 기본 내장 함수

- abs(x) : 절대값 구하기. 정수/실수 모두 지원합니다.
- pow(base, exp, mod=None) : pow(20, 3)은 20^3을 뜻합니다. 만약 세 번째 인자까지 넣으면 pow(4, 2, 3) ➡️ 4^2을 3으로 나눈 나머지(Remainder)인 1을 반환하여 고속 거듭제곱 나머지 연산에 유용합니다.
- divmod(a, b) : (몫, 나머지)를 튜플 형태로 동시에 반환하는 함수입니다. 시/분/초 계산이나 페이지네이션 알고리즘에 자주 쓰입니다.
- max() / min() : 인자들 중 최댓값/최솟값을 찾습니다. 리스트 같은 컨테이너를 넣어도 되고, 문자열을 넣으면 알파벳 순서(ASCII 코드 사전순)로 크기를 비교합니다.
```python
print(abs(-3), abs(-3.8))     # 3, 3.8
print(pow(20, 3), pow(4, 2, 3)) # 8000, 1 (16을 3으로 나눈 나머지)

a, b = divmod(10, 3)          # a=3(몫), b=1(나머지) 언패킹
print(max([10, 20, -2]), min("kim", "kang")) # 20, 'kang' (a가 i보다 사전 앞에 있음)
```

### [5-2] 파이썬 round()의 특이한 작동 메커니즘 (오사오입)

- 오사오입 (Round to nearest even) : 파이썬의 round()는 반올림할 자리의 숫자가 딱 5일 때, 무조건 올리는 것이 아니라 가장 가까운 짝수(Even) 쪽으로 소수점을 맞춥니다. (예: round(3.5) ➡️ 4, round(4.5) ➡️ 4) 금융 및 과학 연산 시 오차를 줄이기 위한 글로벌 표준 방식입니다.
- 음수 자릿수 조절 : round(1236, -2) 처럼 두 번째 인자에 음수를 넣으면 십의 자리에서 반올림하여 백의 자리까지 표현(1200)해 줍니다.
```python
print(round(3.5), round(4.5))      # 출력: 4 4 (둘 다 가장 가까운 짝수인 4로 수렴)
print(round(1236, -2), round(3.745, 1)) # 출력: 1200, 3.7 (45에서 짝수 쪽인 7로 내림됨)
```

## 2. 동적 코드 실행 및 컨테이너 함수

### [5-3] 문자열을 코드로 실행하는 eval()과 exec()

- eval(문자열) : 문자열로 된 파이썬 식(Expression)을 받아 실제 코드로 계산한 뒤 그 결과값을 반환합니다. (예: 텍스트 파일에 적힌 "[1, 2, 3]" 글자를 진짜 리스트 객체로 바꿀 때 유용합니다.)
- 참고: 결과값을 반환하지 않고 a = 10 같은 완전한 문장(Statement)을 실행할 때는 exec()을 사용합니다.
```python
print(eval("1+2"))       # 출력: 3
s = "[10, 20, -5]"
l = eval(s)              # 문자열 글자가 진짜 리스트 객체로 변환됨
print(min(l))            # 출력: -5
```

### [5-4] 컨테이너 전용 내장 함수 (sum과 sorted)

- sum(iterable, start=0) : 컨테이너 내부의 합을 구합니다. 기본적으로 숫자의 합을 구하지만, 두 번째 인자인 start에 빈 튜플()을 지정하면 sum(t2, ()) 연산을 통해 2차원 튜플 구조를 1차원 튜플로 이어 붙여 평탄화(Flatten)하는 트릭이 가능합니다. (단, 문자열 합치기는 지원하지 않아 에러가 납니다.)
- sorted(iterable) : 원본 컨테이너(튜플이든 뭐든)는 절대 건드리지 않고, 정렬된 "새로운 리스트(list)"를 무조건 만들어서 반환합니다.
```python
t1 = (1, 4, 2, 3)
t2 = ((1, 2, 3), (1, 3), (0,))

print(sum(t1, 10))                # 1+4+2+3 에다가 시작값 10을 더함 -> 20
print(sum(t2, ()))                # 2차원 튜플을 합쳐서 1차원으로 만듦 -> (1, 2, 3, 1, 3, 0)
print(type(sorted(t1)), sorted(t1)) # 튜플을 정렬해도 결과물 타입은 무조건 <class 'list'>인 [1, 2, 3, 4]
```

## 3. 시퀀스 메서드 및 실습 알고리즘

### [5-5] 위치 탐색 및 개수 측정 (index, count)

- index(value, start, end) : 찾으려는 값의 방 번호(인덱스)를 반환합니다. 시작점(start)과 끝점(end) 범위를 지정해 "몇 번 방부터 몇 번 방 사이에서 찾아라" 하고 타겟 구간을 좁힐 수 있습니다.
- count(value) : 내부에 해당 데이터가 총 몇 개 들어있는지 개수를 세어줍니다. 없으면 에러 대신 0을 반환합니다.
```python
t = (1, 2, 3, 4, 5, 1, 2, 3)

print(t.index(3))          # 가장 처음 등장하는 3의 위치 -> 2
print(t.index(3, 2))       # 2번 인덱스 '부터' 찾아라 -> 2
print(t.index(3, 3))       # 3번 인덱스 '부터' 찾아라 ➡️ 뒤에 있는 3의 위치인 7 반환
print(t.index(3, 2, 4))    # 2번방부터 4번방 '직전'까지 중 찾아라 -> 2
```

### 🍎 [5-6, 5-7] 실습: 인덱스 매핑 vs 룩업 테이블(Dict) 가격 조회 비교

동일한 조건문 없는 데이터 매핑이지만, 자료구조에 따라 효율성 차이가 크게 납니다.

- [5-6] 튜플 + index() 방식: 이름 튜플에서 인덱스를 뽑아낸 뒤, 가격 튜플의 동일한 인덱스 방을 조회합니다. 이 방식은 index() 함수가 처음부터 방을 다 훑어야 하므로 데이터가 많아지면 느려집니다(O(n)).
- [5-7] 딕셔너리(Dict) 방식: 키-값 쌍으로 묶어두면 컴퓨터가 해시 테이블을 통해 단 한 번에 가격을 바로 매핑해 냅니다(O(1)). 실무에서는 당연히 딕셔너리 방식을 지향해야 합니다.
```python
# [5-6] index 활용 방식
f = ("apple", "banana", "mango", "pineapple", "orange")
p = (1000, 800, 2000, 5200, 400)
print(p[f.index(input())]) # 입력한 과일의 번호표를 따서 가격 조회

# [5-7] 💡 딕셔너리 활용 방식 (훨씬 깔끔하고 압도적으로 빠름)
d = {"apple": 1000, "banana": 800, "mango": 2000, "pineapple": 5200, "orange": 400}
print(d[input()])
```

### [5-8] 대량 데이터 고속 변환기: map() 함수 맛보기

- map(함수, 컨테이너) : 루프(반복문)를 돌리지 않고, 컨테이너 안에 든 모든 알맹이들에 지정한 함수를 동시다발적으로 싹 다 적용시켜 주는 꿀 함수입니다.
- Lazy Evaluation : map 역시 결과를 진짜로 다 만들지 않고 대기하는 map 객체 상태로 리턴되므로, 알맹이를 풀어서 보거나 사용하려면 앞에 애스터리스크()를 붙여 언패킹 출력하거나 list() 등으로 형변환해야 합니다.
```python
f = (3.14, -5.625, 100.4, 25.8)

# 컨테이너 f의 모든 요소에 round 함수를 일괄 적용하라!
m = map(round, f)

print(type(m), m) # 그냥 찍으면 껍데기 정보(<class 'map'>)만 출력됨
print(*m)         # *로 압축 해제(언패킹)하면 반올림된 알맹이들이 튀어나옴 -> 3 -6 100 26
```

## 4. map, zip, enumerate 고급 연동 테크닉

### [5-9] 다중 컨테이너를 받는 map() 함수 응용

- 여러 줄의 컨테이너 동시 가공 : map()은 인자를 2개 이상 받는 함수(pow(base, exp))도 완벽히 지원합니다. 함수 뒤에 두 개 이상의 컨테이너(v, e)를 나열하면, 같은 인덱스 자리에 있는 알맹이들을 하나씩 매칭하여 함수에 주입합니다.
- 동작 매커니즘: pow(10, 2), pow(4, 3), pow(2, 4), pow(-10, 2) 가 순서대로 계산됩니다.
```python
v = (10, 4, 2, -10)
e = (2, 3, 4, 2)

# 두 컨테이너의 요소를 쌍으로 묶어 pow(v, e) 연산을 일괄 적용
print(*map(pow, v, e))  # 출력: 100 64 16 100
```

### [5-10, 5-11] 여러 컨테이너를 하나로 엮는 zip() 함수

- 병렬 데이터 세로로 묶기 : zip()은 여러 개의 시퀀스 객체를 받아, 동일한 인덱스에 위치한 원소들끼리 튜플 묶음으로 엮어주는 지연 평가(Lazy) 함수입니다. 이미지 데이터(RGB 픽셀 묶기)나 행렬 연산에서 필수적으로 사용됩니다.
- 딕셔너리 고속 생성 (dict + zip) : 별개의 튜플로 분리되어 있던 '키 데이터 리스트'와 '값 데이터 리스트'를 zip()으로 묶은 뒤 dict() 감싸주면, 다른 루프 없이 단 한 줄로 완벽한 딕셔너리 매핑 테이블이 완성됩니다.
```python
# [5-10] RGB 이미지 픽셀 매칭 (세로로 묶어서 튜플화)
r = (52, 255, 39, 132)
g = (19, 63, 227, 197)
b = (0, 68, 255, 187)

img = list(zip(r, g, b))
print(img)  # 출력: [(52, 19, 0), (255, 63, 68), (39, 227, 255), (132, 197, 187)]

# [5-11] 두 개의 튜플을 엮어 과일 가격 테이블(Dict) 만들기
f = ("apple", "orange", "banana", "mango")
p = (100, 80, 120, 90)

tag = dict(zip(f, p)) # {'apple': 100, 'orange': 80, ...}
print(tag[input()])   # 과일 이름을 입력하면 매핑된 가격 즉시 출력 (O(1) 속도)
```

### [5-12, 5-13] 값에 번호표를 붙여주는 enumerate() 함수

- zip(range, t) 대신 enumerate : 데이터에 순서대로 일련번호(인덱스)를 매기고 싶을 때, 과거에는 zip(range(len(t)), t)라는 복잡한 방식을 썼지만, 파이썬에서는 이를 완전히 대체하는 내장 함수 enumerate(t)를 제공합니다.
- 시작 번호 커스텀 (start=) : 기본 번호표는 0번부터 시작하지만, enumerate(t, 10) 또는 start=1 옵션을 주면 개발자가 원하는 번호부터 일련번호를 매겨 나가기 시작합니다.
- 자동 인덱스 딕셔너리 생성 : dict(enumerate(f, start=1)) 구조를 쓰면, 품목 리스트를 기반으로 1번부터 자동 증가하는 ID 값을 가진 키-값 사전을 즉시 빌드할 수 있습니다.
```python
# [5-12] 일련번호 매기기 기능 비교
t = ("kim", "lee", "park")

x1 = zip(range(len(t)), t) # 구식 방법 -> (0, 'kim'), (1, 'lee'), ...
x2 = enumerate(t)          # 신식 방법 -> (0, 'kim'), (1, 'lee'), ...
x3 = enumerate(t, 10)      # 10번부터 번호표 부여 -> (10, 'kim'), (11, 'lee'), ...

print(*x2) # 0 'kim' 1 'lee' 2 'park'
print(*x3) # 10 'kim' 11 'lee' 12 'park'


# [5-13] 과일 품목별 자동 번호 부여 및 딕셔너리 변환
f = ("apple", "orange", "banana", "mango")

# 1번부터 인덱스를 매긴 딕셔너리를 단 한 줄로 생성
tag = dict(enumerate(f, start=1))
print(tag) # 출력: {1: 'apple', 2: 'orange', 3: 'banana', 4: 'mango'}
```

## 5. 이터레이터(Iterator)의 소멸 특성과 컨테이너 응용

### ⚠️ [5-14 ~ 5-16] 매우 중요: 이터레이터(Iterator)는 "일회용 빨대"다

- 지연 평가(Lazy Evaluation) 객체의 실체 : map(), zip(), enumerate() 함수가 반환하는 객체들은 메모리에 값을 미리 만들어두지 않는 이름표(Iterator)입니다.
- 일회성 소멸 특성 (Single Pass) : 이 객체들은 '빨대'와 같아서, sum()이나  언패킹 등으로 내부에 들은 알맹이 데이터를 한 번 쭉 빨아먹고 나면 그 즉시 껍데기만 남고 소멸(Empty)합니다.
- 연쇄 소멸 버그 : 따라서 동일한 map 객체를 가지고 두 번 연산(sum(t)을 연속 두 번 실행)하거나, 상위 이터레이터(x)를 먼저 소비해 버리면 하위 이터레이터(y)까지 연쇄적으로 데이터가 증발해 버리는 대참사가 일어납니다. 두 번 이상 데이터가 필요하다면 무조건 list()나 tuple()로 메모리에 완전 박제해 두어야 합니다.
```python
# [5-15] ⚠️ 이터레이터 연속 소비 시 증발 현상
t = map(int, input().split()) # 10 20 30 입력 시
print(sum(t))  # 첫 번째 sum: 빨대로 데이터를 다 빨아먹음 -> 60 출력
print(sum(t))  # 두 번째 sum: 이미 t 내부 데이터가 증발하여 빈 상태 -> 0 출력

# [5-16] ⚠️ 연쇄 증발 버그 분석
x = map(int, [3.14, -5.25, -128]) # [3, -5, -128] 대기 중
y = map(abs, x)                   # x를 재료로 쓰는 하위 빨대 생성

print("[1]", *x) # x를 여기서 먼저 전부 소비해 버림! -> 3 -5 -128 출력
print("[2]", *y) # ⚠️ 재료인 x가 이미 증발해서 y도 빈 깡통이 됨 -> 아무것도 출력 안 됨!
```

### [5-17 ~ 5-18] 데이터 박제(list)와 최댓값 위치 찾기 응용

- map 객체를 list(x)로 박제해 두었기 때문에 max()로 한 번 훑고, index()로 또 한 번 안전하게 재조회할 수 있는 구조입니다.
- lst.append(max_val) 연산을 가해도 원본 리스트의 대표 주소(id)는 고정된 상태로 내부 슬롯만 확장됩니다.
```python
# [5-17] 안전하게 리스트로 박제 후 중복 조회
x = list(map(int, input().split()))
max_val = max(x)
max_pos = x.index(max_val) + 1 # 0번 인덱스 보정
print(max_val, max_pos, sep="\n")

# [5-18] 가변 객체의 원본 수정 연산
lst = list(map(int, input().split()))
max_val = max(lst)
lst.append(max_val) # 원본 리스트 맨 뒤에 최대값 한 통으로 추가
print(max_val, lst, sep="\n")
```

### [5-19 ~ 5-20] 셋(Set)을 통한 중복 제거와 정렬(sorted)의 법칙

- set()의 필터링 기능 : 공백으로 분리된 이름들을 set()에 주입하면 중복된 데이터가 자동으로 파괴됩니다. 그 후 len()을 재면 순수 유니크한 품목 개수만 깔끔하게 떨어집니다.
- 불변 객체와 sorted() : t는 튜플(불변 객체)입니다. 튜플은 자체적으로 자리를 바꾸는 정렬 메서드가 아예 불가능하지만, 내장 함수인 sorted(t)를 사용하면 원본 튜플은 건드리지 않은 채 오름차순/내림차순으로 정렬된 "새로운 리스트(List)"를 이쁘게 반환해 줍니다.
```python
# [5-19] set 자료구조를 이용한 초고속 중복 제거
s = set(input().split()) # 'kim lee kim' 입력 시 {'kim', 'lee'}로 자동 압축
print(len(s))            # 출력: 2

# [5-20] sorted() 함수는 원본 유형에 상관없이 무조건 '새 리스트'를 반환
t = tuple(map(int, input().split()))
```

# 📦 [Chapter 5] 내장 함수 및 시퀀스 메서드 최종 요약

### 1. 파이썬 round()의 특이점 (오사오입)

- 파이썬의 round()는 끝자리가 딱 5일 때 반올림하는 것이 아니라 가장 가까운 '짝수' 쪽으로 붙여버립니다. (round(3.5)는 4, round(4.5)도 4) 십의 자리나 백의 자리 반올림은 자릿수에 음수(2, 3)를 줘서 조절합니다.
### 2. O(n) 인덱싱 vs O(1) 딕셔너리

- 튜플 두 개를 깔아놓고 p[f.index(input())] 형태로 연동하면 맨 처음 방부터 하나씩 다 열어봐야 해서 느려집니다. dict(zip(f, p))로 묶어 테이블을 파놓으면 단 한 번에 가격을 매핑해 내므로 압도적으로 효율적입니다.
### 3. [★초핵심] 이터레이터는 "일회용 빨대"다

- map(), zip(), enumerate()가 뱉어내는 결과물들은 메모리에 값을 들고 있지 않는 임시 이름표입니다.
- sum()이나  언패킹 등으로 알맹이를 한 번 쭉 빨아먹으면 그 즉시 데이터가 증발하여 빈 깡통이 됩니다. 연속으로 두 번 조회하거나 재료 객체를 먼저 써버리면 에러 없이 0개 혹은 빈 칸만 출력되는 연쇄 오염 버그가 납니다. 여러 번 써야 하면 무조건 초반에 list()나 tuple()로 감싸서 메모리에 박제해 두어야 안전합니다.
### 4. sorted()는 무조건 새 리스트를 뱉는다

- 원본이 평생 안 바뀌는 튜플이든 뭐든 간에, 내장 함수 sorted()에 집어넣었다가 나오면 원본은 절대로 건드리지 않은 채 정렬된 완전히 새로운 '리스트(list)'를 무조건 새로 만들어서 반환합니다.
### 5. len()과 .count()의 뇌새김 차이

- len(s) : 바구니 전체를 감싸서 "총 몇 개 들었어?"를 측정 (중복 없는 세트나 딕셔너리도 전부 측정 가능)
- s.count(x) : 바구니 뒤에 점 찍고 꼬리처럼 붙어서 "이 안에 특정 데이터 x가 몇 개 들었어?"를 타겟 조회 (중복을 허용하지 않는 세트set는 애초에 이 메서드가 존재하지 않아 에러 발생)
