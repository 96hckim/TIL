# 🧠 [Chapter 4] 파이썬 메모리 구조와 객체 참조

### [4-1] 컨테이너의 슬롯(Slot) 구조와 객체 재사용

- 슬롯(Slot) 구조 : 파이썬의 리스트(list)는 데이터 자체를 방에 직접 들고 있는 것이 아닙니다. 오직 데이터가 저장된 메모리 주소값(포인터)만 순서대로 담는 연속된 상자(슬롯)들을 가지고 있습니다.
- 주소 공유 (메모리 절약) : 리스트 안의 l[0]과 l[3]은 똑같은 숫자 1입니다. 파이썬은 숫자 1이라는 메모리 객체를 하나만 만들어 두고, 두 슬롯이 동일한 주소값을 가리키게(참조하게) 만듭니다.
```python
l = [1, 2, 3, 1]

# 1. 정수 1, 2, 3이 가진 고유의 메모리 주소(id) 확인
print(id(1), id(2), id(3), "", sep="\n")

# 2. 리스트 자체의 주소, 그리고 리스트 내부 슬롯들이 가리키는 주소 확인
print(*map(id, (l, l[0], l[1], l[2], l[3])), sep="\n")
```

### 🔍 실행 결과 속 주소 분석 (눈여겨볼 포인트)

위 코드를 실행하면 아래와 같은 형태로 메모리 주소(id)들이 출력됩니다.

- id(1)의 주소값과 l[0], l[3]의 주소값이 1의 자리까지 완벽하게 일치하는 것을 확인할 수 있습니다.
- 리스트는 그저 이 주소 번지표들을 차례대로 들고 있는 "주소록 장부"일 뿐입니다.
### [4-2] 문자열(str) 컨테이너의 내부 참조 구조

- 글자 단위 주소 공유: 파이썬은 문자열 "AB0A"를 다룰 때도 리스트와 똑같이 작동합니다. 내부적으로 철자 하나하나를 메모리에 올린 뒤, 그 주소를 슬롯에 차례대로 채워 넣습니다.
- 동일 철자 재사용: 0번 글자인 "A"와 3번 글자인 "A"는 메모리 상에서 완벽히 동일한 주소(id)를 공유합니다.
```python
s = "AB0A"

# 1. 알파벳 글자 객체 고유의 메모리 주소(id) 확인
print(*map(id, "AB0"), "", sep="\n")

# 2. 문자열 전체 주소, 그리고 내부 글자 슬롯들의 주소 확인
print(*map(id, (s, *s)), sep="\n") # s[0]과 s[3]의 주소가 완전히 일치함!
```

### [4-3] 다중 컨테이너의 다차원 주소록 메커니즘

- 주소록 안의 주소록: 리스트 내부에 또 다른 리스트(l[0])나 집합(l[1])이 들어있을 때, 바깥쪽 리스트(l)가 직접 들고 있는 알맹이는 내부 컨테이너의 데이터들이 아니라 내부 컨테이너들의 '시작 주소'입니다.
- len() 함수가 측정하는 것: len(l)을 찍어보면 내부에 데이터가 총 몇 개가 들었든 상관없이, 바깥 바구니가 들고 있는 주소 슬롯의 개수(여기서는 내부 리스트 주소, 집합 주소, 숫자 1 주소 총 3개)인 3만 반환합니다.
```python
# 리스트 내부에 리스트, 집합, 정수가 섞여 있는 다중 구조
l = [[1, 2, 3, 1], {1, 2}, 1]

# 1. 바깥 리스트 주소와 각 슬롯이 가리키는 내부 객체들의 주소 확인
print(*map(id, (l, *l)), sep="\n")

# 2. 컨테이너별 슬롯 개수(길이) 측정
print(*map(len, (l, l[0], l[1]))) # 출력 결과: 3 4 2
# l의 길이는 3 (내부 컨테이너 3개를 가리키는 주소 슬롯이 3개이므로)
# l[0]의 길이는 4 ([1, 2, 3, 1]의 주소 슬롯 개수)
# l[1]의 길이는 2 ({1, 2}의 주소 슬롯 개수)
```

### [4-4] 가변 컨테이너의 아이템 변경과 메모리 주소(id)의 관계

- 바깥 슬롯의 완전 교체 (l[1] = 10) : 1번 슬롯이 바라보던 기존 집합({5, 6})을 끊어버리고 정수 10의 주소로 완전히 갈아 끼웁니다. 당연히 1번 슬롯의 메모리 주소(id(l[1]))는 완전히 다른 값으로 변합니다.
- 안쪽 알맹이의 수정 (l[2][1] = 20) : 2번 슬롯이 가리키는 내부 리스트([7, 8, 9])의 껍데기 주소는 그대로 둔 채, 그 내부의 1번 방 데이터만 슬쩍 바꿉니다. 따라서 내부 요소의 주소는 바뀌지만, 바깥에서 바라보는 내부 리스트의 대표 주소(id(l[2]))는 절대로 변하지 않습니다.
```python
l = [(1, 2, 3, 4), {5, 6}, [7, 8, 9]]

# 1. 수정 전 상태와 주소 기록
print(len(l))
print(*l)
print(id(l[1]), id(l[2][1])) # 기존 {5, 6}의 주소와 숫자 8의 주소 출력

# 2. 데이터 변경 실행
l[1] = 10       # 💡 바깥쪽 슬롯을 정수 10으로 완전 갈아끼움 (주소 바뀜)
l[2][1] = 20    # 💡 안쪽 리스트의 알맹이만 수정 (l[2] 자체의 대표 주소는 고정)

# 3. 수정 후 상태와 주소 비교
print(*l)
print(id(l[1]), id(l[2][1])) # l[1]의 주소는 완전히 바뀌었고, l[2][1]은 20의 주소로 변경됨
```

### [4-5] 가변 컨테이너의 아이템 추가 메커니즘

- l.insert(1, 10) : 바깥쪽 리스트 l에 새로운 주소 슬롯을 하나 비집고 밀어 넣는 연산입니다. 리스트의 슬롯 배열 구조가 변경되므로 len(l)이 3에서 4로 증가합니다.
- l[2].add(20) (⚠️ 예제 코드의 숨겨진 디테일 스파이) : l[2]는 현재 셋째 방에 있는 [7, 8, 9] 리스트를 뜻합니다. 하지만 리스트 객체에는 .add()라는 메서드가 존재하지 않고 .append()나 .insert()를 써야 합니다. (add()는 집합(Set) 전용 메서드입니다.)
```python
# 원본 상태 복구
l = [(1, 2, 3, 4), {5, 6}, [7, 8, 9]]

print(len(l))
print(*l)

l.insert(1, 10) # 바깥 리스트 1번 자리에 10 삽입 (슬롯 한 칸 확장)

# ⚠️ 만약 교재 코드 그대로 l[2].add(20)을 하면 l[2]가 리스트라서 AttributeError가 발생합니다.
# 안전하게 1번 방의 '집합'에 20을 추가하는 코드로 수정하여 실습하는 것을 권장합니다:
l[2].add(20)    # insert로 인해 {5, 6}이 2번 방으로 밀려났으므로 l[2].add()가 정상 작동하게 됩니다! (소름돋는 설계)

print(*l) # 출력 결과: (1, 2, 3, 4) 10 {5, 6, 20} [7, 8, 9]
```

### [4-6] 가변(Mutable) vs 불변(Immutable) 객체의 주소 할당 법칙

- == (동등성): "방 안에 들어있는 알맹이(값)가 같냐?"를 물어봅니다.
- is (동일성): "두 변수가 가리키는 메모리 주소(id)가 완벽히 같냐?"를 물어봅니다.
- 리스트(가변)의 특징: a와 b에 똑같은 [1, 2, 3, 4]를 대입해도, 파이썬은 언제든 값이 바뀔 수 있는 가변 객체 특성을 고려해 메모리에 독립된 별도의 공간 두 개를 팝니다. 따라서 값은 같아도(== True), 주소는 다릅니다(is False).
- 튜플(불변)의 특징: 파이썬 버전에 따라 내부 최적화(CPython의 상수 폴딩 메커니즘 등)가 작동하면, 어차피 평생 안 바뀔 불변 객체인 튜플 d와 e는 메모리 주소를 똑같이 공유하게 만들기도 합니다. (is True가 나올 수 있음)
```python
# --------------------------------------------------
# 1. 가변 객체 (리스트) 테스트
# --------------------------------------------------
a = [1, 2, 3, 4]
b = [1, 2, 3, 4]
c = a  # 💡 주소 복사 (c와 a는 이제 운명공동체)

print(*map(id, (a, b, c)), sep="\n")
# 비교 결과 분석
print(a == b, a == c, a is b, a is c)
# 출력: True True False True
# -> a와 b는 값만 같고 주소는 다름 (is False)
# -> a와 c는 대입 연산자로 주소를 공유했으므로 주소까지 같음 (is True)

# a를 수정하면 주소를 공유하는 c도 함께 바뀜 (b는 독립 공간이라 무사함)
a[0] = 10
print(a[0], b[0], c[0])                # 출력: 10 1 10
print(a == b, a == c, a is b, a is c)  # 출력: False True False True

# --------------------------------------------------
# 2. 불변 객체 (튜플) 테스트
# --------------------------------------------------
d = (1, 2, 3, 4)
e = (1, 2, 3, 4)
f = d

print(*map(id, (d, e, f)), sep="\n")
# 파이썬 인터프리터의 자체 최적화 덕분에 d와 e의 주소(id)가 같게 나올 수 있습니다.
print(d[0], e[0], f[0], d is e, d is f) # 출력: 1 1 1 True True
```

### 💡 요약 한 줄

> ==는 눈에 보이는 '데이터 값'만 비교하고, is는 실제 메모리의 '주소 번지수'를 비교한다. 변수 복사 시 c = a 처럼 그냥 등호로 대입하면 주소값이 복사되어 하나의 데이터가 같이 바뀌는 멀티플 바인딩이 일어난다.

### [4-8] 다중 컨테이너의 결합(+) 및 확장() 연산 시 주소 공유 문제

- 슬롯의 단순 복사: s1 + s2나 s1 * 2 연산을 수행하면 새로운 큰 바구니가 만들어지기는 합니다. 하지만 그 안에 담기는 내용물들은 완전히 새롭게 생성된 리스트가 아니라, 기존 s1과 s2 내부에 있던 하위 리스트들의 메모리 주소를 그대로 복사해 옵니다. (결국 얕은 복사가 일어나는 셈입니다.)
- 연쇄 수정 발생: 따라서 원본인 s1[0][0]의 값을 수정하면, 동일한 하위 리스트의 주소를 가리키고 있던 s3와 s4의 내부 값까지 도미노처럼 동시에 바뀌어 버리는 치명적인 현상이 발생합니다.
```python
s1 = [[1, 2], [3, 4]]
s2 = [[5, 6], [7, 8]]

s3 = s1 + s2
s4 = s1 * 2

# 원본의 아주 깊숙한 알맹이 하나를 수정합니다.
s1[0][0] = 9

# 출력 결과 확인: s1만 바꿨는데 s3, s4의 앞부분도 함께 9로 변해있습니다!
print(s1, s2, s3, s4, sep="\n")

# 풀어서 주소를 찍어보면 s1[0], s3[0], s4[0], s4[2] 등이 모두 같은 id를 바라보고 있습니다.
print(*map(id, (*s1, *s2, *s3, *s4)), sep="\n")
```

### [4-9] [★필수 암기] 2차원 리스트 생성 시 곱셈 연산()의 대참사

- 1차원 리스트 곱셈(a = [0] * 3) : 정수 0은 불변(Immutable) 객체이므로 숫자를 곱해서 공간을 늘려도 아무런 문제가 없습니다. a[0] = 2라고 바꾸면 0번 슬롯이 2라는 정수의 주소로 갈아 끼워질 뿐입니다.
- 2차원 리스트 곱셈(b = [[0]] * 3) : 이 코드는 [[0], [0], [0]] 모양으로 예쁘게 만들어지는 것 같지만, 실체는 하나의 동일한 리스트 [0]의 주소를 3개의 슬롯이 동시에 바라보고 있는 상태입니다. 즉, 3개의 방이 하나의 운명공동체로 묶여있기 때문에 b[0][0] = 3을 하면 세 방이 한꺼번에 [3], [3], [3]으로 변하는 참사가 일어납니다.
```python
a = [0] * 3
b = [[0]] * 3
c = [[0] * 3] * 3

print(a, b, c, sep="\n")

# 값 수정 테스트
a[0] = 2         # 정수는 불변이라 안전하게 0번 방만 2로 바뀜 -> [2, 0, 0]
b[0][0] = 3      # ⚠️ 1개의 주소를 3개 슬롯이 공유하므로 모든 방이 바뀜 -> [[3], [3], [3]]
c[0][0] = 4      # ⚠️ 위와 같은 이유로 3개 행의 0번 열이 동시에 바뀜 -> [[4, 0, 0], [4, 0, 0], [4, 0, 0]]

print(a, b, c, sep="\n")
```

### 🛠️ 현실적인 해결책: 그럼 안전한 2차원 리스트는 어떻게 만들까?

실무나 코딩 테스트에서 값이 공유되지 않는 깨끗한 독립형 2차원 리스트(예: 가로 3, 세로 3)를 만들 때는 곱셈 기호 대신 앞으로 배울 리스트 컴프리헨션(List Comprehension) 구조를 사용하는 것이 정석입니다.

```python
# ✨ 매번 독립된 새로운 리스트 [0, 0, 0]을 3번 생성하라는 안전한 코드
clean_matrix = [[0] * 3 for _ in range(3)]

clean_matrix[0][0] = 7
print(clean_matrix) # 출력: [[7, 0, 0], [0, 0, 0], [0, 0, 0]] -> 0번 방만 깔끔하게 바뀜!
```

## [4-10] 슬라이싱(Slicing)의 고급 작동 메커니즘

- [시작:끝:증감폭] 규칙: 증감폭이 음수(1)이면 뒤에서부터 역순으로 걸어오며 슬라이싱합니다. 역순일 때는 시작점 인덱스가 끝점보다 뒤쪽에 있어야 정상 작동합니다.
- 인덱스 초과 허용: 일반 인덱싱(a[10])은 방 번호가 넘어가면 에러가 나지만, 슬라이싱(b[1:4])은 범위가 실제 데이터 길이보다 길어도 에러 없이 가져올 수 있는 데까지만 쏙 골라 담아주는 안전장치가 있습니다.
```python
a = [0, 1, 2, 3, 4, 5, 6]
b = a[1:3] # [1, 2]

print(a[:], a[2:], a[:5], sep="\n")
print(a[1:-2], a[::2], sep="\n")      # a[::2] -> 건너뛰며 출력: [0, 2, 4, 6]
print(a[::-1], a[:3:-2], sep="\n")    # a[::-1] -> 역순: [6, 5, 4, 3, 2, 1, 0]
print(a[3::-1], a[-4:-1], sep="\n")   # a[3::-1] -> 3번방부터 역순으로: [3, 2, 1, 0]
print(a[:-4:-1], b, b[1:4], sep="\n") # b[1:4] -> 인덱스 범위 초과해도 에러 없이 [2]만 반환
```

### [4-11] 리스트의 연산 및 수정/삭제 메서드 총집합

- += 연산과 extend(): 둘 다 기존 리스트의 껍데기 주소(id)를 유지한 채, 내부 슬롯을 확장하여 이어 붙이는 연산입니다. (문자열 extend('kim')을 하면 'k', 'i', 'm'으로 쪼개져서 추가됩니다.)
- 슬라이싱 대입(l[:] = ...): 리스트의 주소를 유지하면서 내부 알맹이만 단체로 갈아엎는 사기적인 스킬입니다. 단, 간격이 있는 슬라이싱(l[::2] = ...)에 대입할 때는 좌변의 슬롯 개수와 우변의 데이터 개수가 칼같이 일치해야 합니다.
- del l[:]과 clear(): 리스트의 메모리 주소(id)를 깨지 않고, 내부 주소록 슬롯만 싹 지워 빈 리스트[]로 만들어주는 동일한 역할을 합니다.
```python
l = [1, 2]
print(id(l), l, sep = '\n')

l.append((-1, 'lee')) # 튜플 하나를 통째로 추가
l += (5, 'kim')       # 💡 튜플의 알맹이들을 쪼개서 이어붙임 (+= 연산의 특징)
l.extend([7, 8])      # 리스트 이어붙이기
print(id(l), l, sep = '\n') # 원본 주소 유지 확인

l[:] = [1, 2, 3, 4]   # l의 주소는 그대로 두고 알맹이만 [1,2,3,4]로 완전 리셋
print(id(l), l, sep = '\n')

l[1:3:] = [20, 30, 40] # 1번부터 2번 슬롯 자리에 3개의 요소를 비집고 대입
print(id(l), l, sep = '\n')

l[::2] = ('a', 'bc', 'def') # 💡 짝수 칸 개수와 튜플 요소 개수(3개)가 맞아떨어지므로 수정 성공
print(id(l), l, sep = '\n')

del l[1:3]           # 일부 구간 삭제
print(id(l), l, sep = '\n')

del l[:]             # 전체 삭제 (빈 리스트 [] 상태가 됨)
print(id(l), l, sep = '\n')

l.append('lee')      # 'lee' 추가
l.extend('kim')      # 💡 문자열을 쪼개서 'k', 'i', 'm' 각각 추가
print(id(l), l, sep = '\n')

l.reverse()          # 원본 자체를 거꾸로 뒤집기 (주소 고정)
print(id(l), l, sep = '\n')

x = l.copy()         # 얕은 복사 (새 주소)
y = l[::-1]          # 슬라이싱으로 뒤집기 (새 주소 반환)
print(x, y, sep = '\n')

l.clear()            # 메모리 주소 유지하며 완전 초기화
print(id(l), l, sep = '\n')
```

### [4-12] 객체 복사의 3단계 (메서드 vs copy 모듈)

- .copy()와 copy.copy(): 동일하게 작동하는 얕은 복사입니다. 1차원 주소록은 분리되지만, 내부의 2차원 리스트 주소(id(l1[0]))는 동일하게 공유됩니다.
- copy.deepcopy(): 내부의 하위 리스트 주소까지 완벽하게 추적해서 새 공간에 따로 만들어내는 깊은 복사입니다. 원본 l1을 훼손해도 l4는 혼자 무사합니다.
```python
import copy

l1 = [[1, 2], [3, 4]]
l2 = l1.copy()         # 얕은 복사 1안
l3 = copy.copy(l1)     # 얕은 복사 2안
l4 = copy.deepcopy(l1) # ✨ 깊은 복사 (완벽한 독립)

l1[0][0] = 10          # 원본의 안쪽 알맹이 수정

# 주소(id)와 출력 확인해보면 l1, l2, l3는 다 같이 오염되었지만, l4 혼자 깨끗하게 보존됩니다.
print(id(l1[0]), l1, sep='\n')
print(id(l2[0]), l2, sep='\n')
print(id(l3[0]), l3, sep='\n')
print(id(l4[0]), l4, sep='\n') # 오직 l4의 하위 리스트만 id가 다른 것 확인!
```

### [4-13] 포함 여부를 검사하는 멤버십 연산자 (in / not in)

- 내부에 특정 데이터가 존재하는지 참/거짓으로 알려줍니다.
- 딕셔너리의 기본 탐색 대상: 딕셔너리에 대고 그냥 in 연산을 돌리면 Key만 뒤집니다. Value나 쌍 전체를 찾고 싶다면 반드시 뒤에 .values() 나 .items() 객체를 명시해 줘야 합니다.
```python
t = ([1, 2, 3, 1], {1, 2}, (1, 2, 3))
l = [[1, 2, 3, 1], {1, 2}, (1, 2, 3)]
s = {1, 3, 5}
d = {'a': 1, 'b': 2, 'c': 3}
r = range(1, 5)

print({1, 2} in t, (1, 3) in l, 3 not in s, 'a' in d, 3 in r) # True False False True True

# 💡 딕셔너리 정밀 탐색 테크닉
print(('a', 1) in d.items(), 2 in d.values(), 'c' in d.keys()) # True True True
```

# 🧠 Chapter 4. 파이썬 메모리와 참조 구조 핵심 요약

### 1. 컨테이너의 실체: "주소록 장부"

- 파이썬의 리스트, 튜플, 문자열 등 컨테이너는 데이터 자체를 방에 담지 않습니다.
- 데이터가 들어있는 메모리 주소(ID) 포인터들만 순서대로 나열한 슬롯(Slot) 구조입니다.
- 동일한 글자나 정수(예: 1, "A")는 새로 메모리를 파지 않고 기존 객체의 주소를 공유하여 메모리를 극도로 아낍니다.
### 2. 가변 객체의 주소 변동 규칙

- l[1] = 10 처럼 바깥쪽 슬롯을 아예 다른 데이터로 대입하면 주소(id)가 교체됩니다.
- l[2][1] = 20 이나 l.append() 처럼 내부 알맹이를 수정/추가할 때는 바깥쪽 컨테이너의 대표 주소(id(l))는 절대로 변하지 않고 유지됩니다.
### 3. 연산자 비교: == vs is

- == (동등성): 두 객체의 "값(Value)"이 같으면 True입니다.
- is (동일성): 두 객체의 "메모리 주소(id)"가 완벽히 일치해야 True입니다.
- c = a 처럼 변수를 그냥 등호로 대입하면 멀티플 바인딩(주소 복사)이 일어나며, 두 변수는 하나를 바꾸면 같이 바뀌는 운명공동체가 됩니다.
### 4. 객체 복사 버그와 대참사

- 얕은 복사 (.copy(), [:]): 1차원 바구니만 분리되고 내부의 2차원 중첩 컨테이너 주소는 공유하므로 연쇄 오염 위험이 있습니다.
- 깊은 복사 (copy.deepcopy()): 중첩 구조까지 완벽 추적하여 완전히 독립된 새 공간에 복사합니다.
- 2차원 리스트 생성 주의: [[0]] * 3 처럼 곱셈 연산으로 2차원 리스트를 만들면 내부의 모든 행이 하나의 주소를 공유하는 대참사가 납니다. 안전한 다차원 배열은 무조건 리스트 컴프리헨션([[0] * 3 for _ in range(3)])으로 파야 합니다.
### 5. 멤버십 연산의 타깃

- in 연산자 사용 시 딕셔너리는 기본적으로 Key만 뒤집니다. Value나 세트를 찾으려면 .values()와 .items()를 꼭 활용합시다.
