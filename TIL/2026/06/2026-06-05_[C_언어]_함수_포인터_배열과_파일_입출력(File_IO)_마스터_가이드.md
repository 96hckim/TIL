[함수 포인터 배열]

## 1. 🗺️ 한눈에 보는 메모리 구동 원리

일반 포인터 배열이 메모리의 데이터(RAM) 주소를 가리킨다면, 함수 포인터 배열은 컴퓨터가 실행할 명령어들이 저장된 코드(Code) 영역의 주소를 가리킵니다.

- 원리: 배열의 인덱스 번호([0], [1], [2])가 곧 함수 명세서 번호가 됩니다. 배열이름[인덱스](매개변수); 기호 하나로 복잡한 분기문 없이 원하는 함수를 다이렉트 워프시켜 실행합니다.
## 2. 💻 함수 포인터 배열 공식 가이드

함수 포인터 배열을 선언할 때는 묶으려는 함수들의 "리턴 타입"과 "매개변수 형태"가 완벽하게 일치해야 체인으로 묶을 수 있습니다.

```c
// 💡 기본 공식: 리턴타입 (*배열이름[]) (매개변수타입1, 매개변수타입2...)

// 예시: 리턴이 void이고, 인자로 (Account_t*, int*)를 받는 함수들의 배열
void (*menu_funcs[])(Account_t*, int*) = {
    create_account,       // index 0
    deposit,              // index 1
    withdraw,             // index 2
    display_all_accounts  // index 3
};
```

### ⚠️ 주의할 점 (괄호의 중요성)

- void (*menu_funcs[])() : 함수 포인터들의 배열 (우리가 원하는 것 ⭕)
- void *menu_funcs[]() : 주소(void*)를 리턴하는 일반 함수들의 배열 (완전 오작동 ❌)
## 🛠️ 3. switch문 vs 함수 포인터 배열 구조 비교

메뉴 분기 처리를 할 때 두 방식의 코드 다이어트 차이입니다.

### 🧱 과거: switch~case 방식 (늘어지는 코드)

```c
switch (choice) {
    case 1: create_account(acc, &count); break;
    case 2: deposit(acc, &count); break;
    case 3: withdraw(acc, &count); break;
    case 4: display_all_accounts(acc, &count); break;
}
```

### 💎 진화: 함수 포인터 배열 방식 (단 한 줄)

```c
// 인덱스 범위 예외 처리만 앞에 해두면 단 한 줄로 깔끔하게 끝납니다.
menu_funcs[choice - 1](acc, &count);
```

## 📊 함수 포인터 배열의 장단점 요약표

| 👍 장점 (장착해야 하는 이유) | 👎 단점 및 주의점 |
|---|---|
| 압도적인 가독성: 수백 줄짜리 switch 문을 단 한 줄로 축약 | 엄격한 규칙: 묶으려는 함수들의 리턴형/매개변수가 똑같아야 함 |
| O(1)의 고속 분기: 순차 대조가 아닌 주소 다이렉트 점프 연산 | 가독성 난해: 초보 개발자가 처음 볼 때 문법이 어려워 보임 |
| 확장성 업: 메뉴가 추가되어도 main 로직 수정 없이 배열만 확장 | 인덱스 오버플로우: 잘못된 인덱스 참조 시 팅김 위험 (예외처리 필수) |

### 📋 한 줄 요약

> 함수 포인터 배열은 동일한 규격(리턴형, 매개변수)을 가진 함수들의 주소를 모아놓은 **'함수 실행 테이블'**이며, 이를 활용하면 복잡하고 거대한 switch~case 분기문을 지워버리고 단 한 줄의 인덱스 참조만으로 함수를 초고속 호출하는 고급 아키텍처를 구현할 수 있다!



## 1. fopen (File Open) : 파일 열기

파일을 사용하기 위해 운영체제로부터 파일의 주소록(스트림 포인터)을 얻어오는 함수입니다.

### 📌 함수 원형

```c
FILE* fopen(const char* filename, const char* mode);
```

- 반환값: 성공 시 파일의 주소를 가리키는 FILE* 포인터 반환, 실패 시 NULL 반환.
- 사용 예시:
```c
FILE* fp = fopen("test.txt", "w");
if (fp == NULL) {
    printf("파일 열기 실패!\n");
}
```

### 📌 가장 많이 쓰는 파일 모드 (Mode) 주주 정리

| 모드 | 의미 | 파일이 없을 때 | 파일이 이미 있을 때 |
|---|---|---|---|
| "r" (Read) | 읽기 전용 | 에러 발생 (NULL 반환) | 파일의 처음부터 읽음 |
| "w" (Write) | 쓰기 전용 | 새로 생성 | 기존 내용 전부 삭제 (덮어쓰기) |
| "a" (Append) | 이어 쓰기 | 새로 생성 | 기존 내용 뒤에 이어서 작성 |

> 💡 바이너리 파일(이미지, 데이터 등)을 다룰 때 팁

## 2. fclose (File Close) : 파일 닫기

사용이 끝난 파일의 연결을 끊고, 메모리 버퍼를 완전히 비우는 함수입니다.

### 📌 함수 원형

```c
int fclose(FILE* stream);
```

- 반환값: 성공 시 0 반환, 실패 시 EOF(-1) 반환.
- 사용 예시:
```c
fclose(fp);
```

### 📌 닫지 않으면(fclose를 안 하면) 생기는 문제점

1. 데이터 유실: 파일에 글을 쓸 때, 프로그램은 즉시 하드디스크에 저장하지 않고 임시 메모리(버퍼)에 모아둡니다. fclose를 해야 이 버퍼가 파일로 완전히 물리적 저장이 되는데, 안 닫고 강제 종료되면 내용이 날아갈 수 있습니다.
1. 메모리 누수 (Resource Leak): 파일 포인터가 차지하는 시스템 자원이 계속 낭비됩니다.
1. 파일 잠김: 프로그램이 켜져 있는 동안 다른 프로그램에서 해당 파일을 수정하거나 지우지 못하게 잠겨버릴 수 있습니다.
## 📝 한 줄 요약

- fopen은 파일을 여는 함수이며, 실패할 확률(NULL)이 있으므로 항상 예외 처리가 필요하다.
- fclose는 버퍼를 비우고 자원을 반환하는 함수이며, fopen을 했다면 반드시 세트로 작성해야 한다.
- 텍스트는 "r", "w", "a", 바이너리는 "rb", "wb", "ab"를 쓴다.


## 1. fgetc (File Get Character) : 파일에서 한 글자 읽기

파일 포인터가 가리키는 위치에서 문자 딱 하나를 읽어오고, 파일 포인터의 위치를 자동으로 다음 문자로 1칸 이동시킵니다.

### 📌 함수 원형

```c
int fgetc(FILE* stream);
```

- 반환값: * 성공 시: 읽어온 문자(ASCII 코드 값)를 반환합니다.
- 특이점 (중요): 반환 타입이 char가 아니라 int입니다. 파일의 끝을 알리는 EOF 값이 1이기 때문에, 이를 정확하게 표현하고 처리하기 위해서 int형으로 반환합니다.
### 📌 사용 예시 (파일 끝까지 한 글자씩 읽어 출력하기)

```c
int ch; // EOF(-1)를 받기 위해 int형으로 선언
FILE* fp = fopen("test.txt", "r");

if (fp != NULL) {
    // 파일 끝(EOF)을 만날 때까지 무한 반복하며 한 글자씩 읽음
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch); // 읽어온 한 글자를 화면에 출력
    }
    fclose(fp);
}
```

## 2. fputc (File Put Character) : 파일에 한 글자 쓰기

파일 포인터가 가리키는 위치에 문자 딱 하나를 집어넣고, 포인터를 1칸 이동시킵니다.

### 📌 함수 원형

```c
int fputc(int character, FILE* stream);
```

- 매개변수:
- 반환값: * 성공 시: 방금 쓴 문자를 그대로 반환합니다.
### 📌 사용 예시 (파일에 'A'부터 'Z'까지 쓰기)

```c
FILE* fp = fopen("alphabet.txt", "w");

if (fp != NULL) {
    // 'A'부터 'Z'까지 한 글자씩 파일에 기록
    for (char ch = 'A'; ch <= 'Z'; ch++) {
        fputc(ch, fp);
    }
    fclose(fp); // 결과 파일 내용: ABCDEFGHIJKLMNOPQRSTUVWXYZ
}
```

## 📝 한 줄 요약

- fgetc와 fputc는 파일 입출력 함수 중 가장 지름길인 "1바이트(한 글자) 전용 통로"이다.
- fgetc는 파일의 끝에 도달하면 EOF(-1)를 반환하므로, 받아주는 변수는 char가 아니라 반드시 int형으로 선언해야 안전하다.
- 함수 이름 뒤의 c는 Character(문자)를 뜻한다! (fgetc = File Get Character / fputc = File Put Character)


## 1. 개방 모드 (File Open Mode)

fopen 함수를 사용할 때 파일의 사용 목적을 지정하는 문자열입니다. 기본 모드(r, w, a)에 기능이 추가되는 형태로 기억하시면 쉽습니다.

- r (Read): 읽기 전용. 파일이 없으면 에러(NULL)가 발생합니다.
- w (Write): 쓰기 전용. 파일이 없으면 만들고, 이미 있으면 내용이 싹 지워진 채로 새로 씁니다.
- a (Append): 이어 쓰기. 파일이 없으면 만들고, 있으면 기존 내용 맨 뒤에 이어서 씁니다.
### ➕ 확장 모드 (더 정밀한 제어)

- + (수정 모드): 읽기와 쓰기를 동시에 가능하게 만듭니다.
- b (바이너리 모드): 이미지, 구조체, 실행 파일 등 이진 데이터를 다룰 때 붙입니다.
## 2. fseek : 파일 포인터 위치 마음대로 이동하기

파일을 읽거나 쓸 때, 현재 위치(Cursor)를 내가 원하는 바이트 수만큼 강제로 이동시키는 함수입니다.

### 📌 함수 원형

```c
int fseek(FILE* stream, long offset, int origin);
```

- offset (이동 거리): 몇 바이트 이동할지 결정합니다. 양수(+)면 오른쪽(뒤로), 음수()면 왼쪽(앞으로) 이동합니다.
- origin (기준점): 어디를 기준으로 움직일지 결정하는 세 가지 상수입니다.
### 📌 사용 예시

```c
fseek(fp, 0, SEEK_END);      // 파일의 맨 끝으로 이동 (파일 크기 잴 때 많이 씀)
fseek(fp, -10, SEEK_CUR);    // 현재 위치에서 앞으로(왼쪽으로) 10바이트 이동
fseek(fp, sizeof(int), SEEK_SET); // 맨 처음에서 int 크기만큼 뒤로 이동
```

## 3. rewind : 파일 포인터를 맨 처음으로 리셋

말 그대로 비디오테이프를 뒤로 감듯, 파일 포인터를 파일의 맨 첫 번째 위치로 되돌리는 함수입니다.

### 📌 함수 원형

```c
void rewind(FILE* stream);
```

- fseek(fp, 0, SEEK_SET);을 수행한 것과 완벽하게 동일하게 동작합니다.
- 파일에 데이터를 다 쓰고 나서 처음부터 다시 읽어야 할 때 아주 자주 쓰이며, 코드가 직관적이어서 자주 애용됩니다.
## 4. feof : 파일의 끝(End Of File) 체크하기

현재 파일 포인터가 파일의 맨 끝(더 이상 읽을 데이터가 없는 곳)에 도달했는지 확인하는 함수입니다.

### 📌 함수 원형

```c
int feof(FILE* stream);
```

- 반환값: 아직 끝이 아니면 0, 파일의 끝에 도달했으면 0이 아닌 값(참)을 반환합니다.
### ⚠️ 주의점 (가장 많이 하는 실수!)

feof는 파일 끝을 미리 예측하는 함수가 아닙니다. "이미 파일 끝을 지나쳐서 읽으려고 시도했다가 실패한 적이 있는가?"를 물어보는 함수입니다.

따라서 아래와 같이 작성하면 마지막 데이터가 두 번 출력되는 버그가 생길 수 있습니다.

```c
// ❌ 잘못된 예시 (마지막 글자가 두 번 찍힐 위험 있음)
while(!feof(fp)) {
    ch = fgetc(fp);
    printf("%c", ch);
}

// ⭕ 올바른 예시 (읽기 시도를 먼저 하고 성공 여부를 체크)
while(1) {
    ch = fgetc(fp);
    if(feof(fp)) break; // 읽어봤더니 끝이었으면 탈출!
    printf("%c", ch);
}
```

## 📝 한 줄 요약노트
