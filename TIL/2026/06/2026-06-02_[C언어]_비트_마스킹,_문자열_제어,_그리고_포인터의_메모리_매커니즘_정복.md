1. 비트 제어 핵심 공식 (Bit Masking)
특정 위치(n번)의 비트를 켜고 끄는 실무 표준 패턴입니다.


• 비트 끄기 (Reset): c &= ~(1 << n)
    ◦ 1 << 31로 31번 비트만 1로 만든 뒤, ~로 뒤집어 31번만 0이고 나머지는 1인 마스크를 만듭니다. 이를 & 연산하여 31번 비트만 핀포인트로 꺼버립니다.


• 비트 켜기 (Set): c |= (1 << n)
    ◦ 1 << 31로 31번 비트만 1인 마스크를 만든 뒤, | 연산하여 다른 비트는 보존하고 31번 비트만 강제로 1로 켭니다.


2. 이진수 출력 함수 원리 요약

```c
void print_binary(int num)
{
    // 7번(왼쪽)부터 0번(오른쪽) 비트까지 거꾸로 순회
    for (int i = 7; i >= 0; i--)
    {
        // [원리] i칸 밀어서(>> i) 맨 끝자리 비트만 필터링(& 1)
        int bit = (num >> i) & 1;
        printf("%d", bit);

        // 가독성: 4비트마다 공백 출력 (단, 맨 마지막 0번에서는 제외)
        if (i % 4 == 0 && i != 0)
        {
            printf(" ");
        }
    }
    printf("\n");
}
```

[포인터]

### 1. 포인터의 본질 (개념 정의)

- 일반 변수: 숫자(10)나 문자('A') 같은 실제 데이터를 저장하는 그릇입니다.
- 포인터 변수: 다른 변수가 메모리 상에서 몇 번지에 살고 있는지, 그 주소값을 저장하는 전용 그릇입니다.
### 2. 포인터를 왜 사용할까? (가장 큰 이유)

- 메모리 직접 참조 (Access): C언어에서 서로 다른 함수(예: main 함수와 내가 만든 사용자 정의 함수)는 자기 방에 있는 지역 변수를 서로 들여다볼 수 없습니다.
하지만 원본 변수의 주소(포인터)를 넘겨주면, 호출된 함수에서 외부 함수에 있는 변수 값을 직접 읽거나 수정(Access)할 수 있게 됩니다. 이를 통해 메모리를 낭비하지 않고 효율적으로 데이터를 제어할 수 있습니다.
### 3. 포인터 선언 및 주소 저장 방법

포인터를 선언할 때는 가리키고자 하는 원본 변수의 데이터 타입 뒤에 별표(*)를 붙입니다. 변수 이름은 포인터임을 알아보기 쉽게 맨 앞에 소문자 p를 붙이는 카멜 케이스 규칙을 주로 씁니다.

```c
int age = 20;     // 일반 변수 선언
int* pAge;        // int형 주소를 담을 포인터 변수 pAge 선언

pAge = &age;      // 주소 연산자(&)를 사용해 age의 메모리 주소를 포인터에 저장
```

### 4. 포인터 핵심 연산자 기호 2가지

포인터를 지배하는 기호는 딱 두 가지입니다. 돋보기(&)와 화살표(*)로 기억하시면 쉽습니다.

- & (주소 연산자): "너 어디 사니?"
- * (역참조 연산자 / 간접 참조 연산자): "그 주소로 가라!"
```c
printf("%p\n", pAge);  // age의 주소값이 출력됨 (예: 00xHEX...)
printf("%d\n", *pAge); // pAge가 가리키는 주소로 찾아가서 실제 값인 20을 출력함
```

### 💡 한 줄 요약

> 변수는 데이터를 담고, 포인터는 그 데이터가 있는 주소를 담는다! 함수 간에 주소(&)를 주고받음으로써 다른 함수에 있는 원본 변수를 자유롭게 읽고 쓸(*) 수 있게 된다.

```c
/**
 * [개념 정리]
 * 1. 변수(Variable): 데이터를 담는 메모리 공간(그릇)
 * 2. 포인터(Pointer): 다른 메모리 공간의 '주소'를 담는 전용 공간
 * 3. 포인터 사용의 본질:
 * 호출된 함수에서 외부 함수(예: main)의 지역 변수에 직접 접근(Read/Write)하여
 * 메모리를 효율적으로 공유하고 원본 데이터를 제어하기 위함이다.
 */

#define ALPHABET_COUNT (26)

void shift_alpha(char* pAlpha);
void pointer_shift(char* pAlphaArray);

int main(void)
{
	// 뒤에 자동으로 \0이 붙으므로 총 27칸짜리 배열이 됩니다.
	char alpha[] = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";

	printf("원본 문자열 ==> %s\n\n", alpha);

	// 배열의 이름은 첫 번째 방의 시작 주소(alpha[0])를 의미합니다.
	//shift_alpha(alpha);
	pointer_shift(alpha);

	printf("\n");
	printf("alpha[0]의 주소: %p\n", (void*)alpha);
	printf("alpha[1]의 주소: %p\n", (void*)&alpha[1]);

	return 0;
}

/**
 * @brief 알파벳 배열을 한 칸씩 밀어가며 회전판 형태로 출력하는 함수
 * @param pAlpha 알파벳 문자열 배열의 시작 주소
 */
void shift_alpha(char* pAlpha)
{
	for (int i = 0; i < ALPHABET_COUNT; i++)
	{
		for (int j = 0; j < ALPHABET_COUNT; j++)
		{
			printf("%c", pAlpha[(i + j) % ALPHABET_COUNT]);
		}
		printf("\n");
	}
}

/**
 * @brief 포인터 이동 연산만을 사용하여 문자열 자체를 1칸씩 왼쪽으로 회전시키는 함수
 * @param pAlphaArray 알파벳 문자열 배열의 시작 주소
 */
void pointer_shift(char* pAlphaArray)
{
	char temp_char;
	char* pCurrent;

	for (int i = 0; i < ALPHABET_COUNT; i++)
	{
		// 1. 매 회전마다 포인터 위치를 배열의 시작점(0번 방)으로 초기화
		pCurrent = pAlphaArray;

		// 2. 현재 상태의 문자열 출력 (널 문자가 안전하게 보존되어 있으므로 %s 가능)
		printf("%s\n", pCurrent);

		// 3. 맨 앞 글자(0번 방)를 변수에 임시 대피
		temp_char = *pCurrent;

		// 4. 포인터 연산으로 뒤의 글자를 앞방으로 한 칸씩 당기기
		for (int j = 0; j < ALPHABET_COUNT - 1; j++)
		{
			*pCurrent = *(pCurrent + 1); // 다음 방의 값을 현재 방에 대입
			pCurrent++;                  // 다음 주소로 포인터 이동
		}

		// 5. 당기기가 끝나면 pCurrent는 마지막 방(25번 인덱스)에 도착해 있습니다.
		// 대피시켜 두었던 첫 글자를 이 마지막 방에 골인시킵니다.
		*pCurrent = temp_char;
	}
}
```

### 🔄 주소 연산자(&)와 역참조 연산자(*)의 역함수(상쇄) 관계

수학에서 더하기와 빼기, 곱하기와 나누기가 서로를 상쇄시켜 원래 숫자로 되돌리는 것처럼, C언어의 & (주소 구하기)와 *** (주소로 찾아가기)도 완벽한 역함수 관계입니다.

따라서 두 기호가 연속으로 붙으면 서로의 효과를 완전히 없애버리고 원본 변수 자기 자신으로 돌아옵니다.

### 💻 코드로 보는 상쇄 현상

```c
int a = 10;
```

- &a: 변수 a의 주소값을 구합니다. (예: 100번지)
- &a: &a로 구한 주소(100번지)로 다시 찾아갑니다.
결국 "a의 주소를 구한 뒤, 다시 그 주소로 가라"가 되기 때문에, *와 &가 짝을 이뤄 펑 사라지고 그냥 a와 완벽히 똑같은 상태가 됩니다.

```c
printf("%d\n", *&a); // 'a'를 그냥 찍은 것과 똑같이 10이 출력됨
*&a = 30;            // 'a = 30;' 규칙과 완벽히 동일하게 변수 값이 바뀜
```

### 📋 한 줄 요약

> *와 &는 서로 반대로 작동하는 역함수 관계이기 때문에, 변수 앞에 두 기호가 동시에 붙은 *&변수는 기호가 전부 상쇄되어 그냥 변수 원본 자체가 된다!



### 1. L-value(좌변)와 R-value(우변)의 진짜 의미

많은 입문자가 단순히 대입 연산자(=)의 왼쪽/오른쪽 위치로만 기억하지만, 컴파일러 기준의 진짜 의미는 다음과 같습니다.

- L-value (Left-value / 좌변)
- R-value (Right-value / 우변)
### 2. *와 & 연산자의 L-value / R-value 규칙

### ❶  *(역참조 변수) ➡️ L-value, R-value 둘 다 가능! (O)

포인터 앞에 *를 붙인 *p는 주소문을 열고 들어간 "실제 원본 변수의 메모리 공간"을 의미하기 때문에 둘 다 가능합니다.

```c
int a = 10;
int b = 0;
int* p = &a;

*p = 20;   // [L-value 사용] a의 메모리 공간에 20을 집어넣음 (좌변)
b = *p;    // [R-value 사용] a 공간에 있던 값(20)을 꺼내서 b에 대입 (우변)
```

### ❷ & (주소 연산자) ➡️ 오직 R-value만 가능! (O)

변수의 주소를 구한 결과(예: &a)는 컴퓨터가 그 순간 계산해 낸 임시 숫자(주소 상수 값)일 뿐입니다. 이 주소 숫자 자체가 값을 담을 수 있는 변수 공간은 아니기 때문에 좌변에 올 수 없습니다.

```c
int a = 10;
int* p;

p = &a;    // [R-value 사용] 주소값 '100번지'라는 숫자를 p에 대입 (우변)
&a = 500;  // [오류! L-value 불가능] "100번지라는 숫자 자체에 500을 대입하라"가 되므로 컴파일 에러!
```

### 💡 노션 복습용 최종 확장판 요약

> **L-value**는 실제 주소가 있는 **지속적인 메모리 공간(좌변 가능)**이고, **R-value**는 쓰고 사라지는 **임시 값(우변만 가능)**이다.***p**는 주소로 찾아간 실제 변수 공간이므로 L-value와 R-value 모두 쓰일 수 있다.**&a**는 임시로 계산된 **주소 숫자(상수)**일 뿐이므로 오직 R-value로만 쓰일 수 있다.



[배열과 포인터]

### 1. 배열명은 첫 번째 요소의 주소다.

- 예시: arr이라는 이름만 단독으로 쓰면 &arr[0]과 완벽히 똑같은 1000번지입니다.
```c
int arr[3] = {10, 20, 30};

printf("%p\n", arr);        // 출력: 1000 (시작 주소)
printf("%p\n", &arr[0]);     // 출력: 1000 (0번 방의 주소로 동일)
```

### 2. 포인터에 배열명을 저장하면 포인터를 배열명처럼 사용할 수 있다.

- 예시: 포인터 변수 p에 arr을 대입하면, 대괄호([])를 붙여 p[1] 같은 배열 문법을 그대로 쓸 수 있습니다.
```c
int arr[3] = {10, 20, 30};
int* p = arr;               // 포인터에 배열 이름(주소)을 저장

printf("%d\n", p[1]);       // 출력: 20 (arr[1]과 똑같이 동작함)
```

### 3. 배열명의 정수 덧셈은 가리키는 자료형의 크기를 곱해서 더한다.

- 예시: int형은 4바이트이므로, 주소에 1을 더하면 실제 메모리 번지는 4바이트가 더해진 1004번지가 됩니다.
```c
int arr[3] = {10, 20, 30};

// 주소 계산 공식: 시작 주소(1000) + (정수 1 * int 크기 4바이트) = 1004번지
printf("%p\n", arr + 1);    // 출력: 1004
printf("%d\n", *(arr + 1)); // 출력: 20 (1004번지에 사는 값)
```

### 4. 포인터의 뺄셈 결과는 배열 요소 간의 간격 차이를 의미한다.

- 예시: 2번 방 주소(1008번지)에서 0번 방 주소(1000번지)를 빼면, 바이트 차이(8)가 아니라 "몇 칸 떨어져 있는가(2)"가 나옵니다.
```c
int arr[3] = {10, 20, 30};
int* p2 = &arr[2];          // 2번 방 주소 (1008번지)
int* p0 = &arr[0];          // 0번 방 주소 (1000번지)

// 간격 계산 공식: (1008 - 1000) / int 크기 4바이트 = 2칸
printf("%d\n", p2 - p0);    // 출력: 2 (두 포인터 사이에 방이 2개 차이 난다는 뜻)
```

### 💡 요약

