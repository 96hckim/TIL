## 2. 마이크로컨트롤러(MCU) vs 마이크로프로세서(MPU)

두 장치는 모두 시스템의 '두뇌' 역할을 하지만, 칩의 구성 방식과 사용 목적에서 큰 차이가 있습니다.

### 📊 한눈에 비교하는 핵심 차이점

| 구분 | 마이크로컨트롤러 (MCU) | 마이크로프로세서 (MPU) |
|---|---|---|
| 핵심 개념 | 하나의 칩에 컴퓨터 부품을 모두 넣음 (원칩) | 고성능 연산 장치(CPU) 기능에만 집중 |
| 칩의 구성 | CPU + RAM + ROM + I/O 단자 가 내장 | RAM, ROM, I/O 단자를 외부에 따로 연결해야 함 |
| 주요 용도 | 가전제품, 자동차 제어, IoT 기기, 로봇 | PC, 스마트폰, 서버, 고성능 그래픽 처리 |
| 성능 (클럭) | 비교적 낮음 (몇 MHz ~ 수백 MHz) | 매우 높음 (몇 GHz 이상) |
| 운영체제(OS) | OS가 없거나 가벼운 RTOS 사용 | Windows, Linux, Android 등 대형 OS 필수 |
| 가격 및 전력 | 저렴하고 소비 전력이 매우 낮음 | 비싸고 소비 전력이 높음 (발열 관리 필요) |
| 대표 예시 | ATmega128A, Arduino, STM32 | Intel Core 시리즈, AMD Ryzen, Apple M 시리즈 |

### 🔍 조금 더 자세히 알아보기

📢 1. 칩 구성 방식의 차이 (원칩 vs 멀티칩)

- MCU: 방 하나에 침대(RAM), 책상(ROM), 컴퓨터(CPU)가 다 들어있는 '원룸'과 같습니다. 공간은 좁지만 동선이 짧고 효율적입니다.
- MPU: 거실에는 CPU가 있고, 안방에는 RAM이 있고, 건너방에는 SSD가 있는 '큰 아파트'와 같습니다. 성능은 뛰어나지만 구조가 복잡하고 주변 부품들이 반드시 같이 있어야 작동합니다.
📢 2. 실행하는 소프트웨어의 차이

- MCU: 전원이 켜지면 내가 C언어로 작성해서 집어넣은 단 하나의 main 루프(무한 반복문)만 실행합니다. 윈도우 같은 화면이 뜰 필요 없이 바로 작동해야 하기 때문입니다.
- MPU: 컴퓨터를 켜면 Windows나 Linux 같은 무거운 운영체제(OS)를 먼저 메모리에 올린 뒤, 그 위에서 인터넷 브라우저, 게임, 문서 프로그램 등 여러 프로그램을 동시에(멀티태스킹) 실행합니다.


# 📊 ATmega128 vs Intel Core i7 스펙 비교

| 항목 | 마이크로컨트롤러 (ATmega128) | 데스크톱 컴퓨터 (Intel Core i7) |
|---|---|---|
| CPU | ATmega128 | 인텔 Core i7 |
| 비트 (Bit) | 8비트 | 64비트 |
| 메모리 (Memory) | 128 KB (플래시 메모리) | 8 GB (RAM) |
| 클록 (Clock) | 16 MHz | 3.4 GHz (쿼드 코어) |

## 🔍 각 항목별 상세 설명

### 1. 비트 (Bit): 8비트 vs 64비트

> 💡 데이터 처리 도로의 폭 (차선 수)

- 8비트 (ATmega128): CPU가 한 번에 처리할 수 있는 데이터의 크기가 8비트(2^8 = 256가지 상태)입니다. 숫자로 치면 0부터 255까지의 숫자만 한 번에 처리할 수 있습니다.
- 64비트 (Core i7): 한 번에 64비트라는 어마어마한 양의 데이터를 처리합니다.
### 2. 메모리 (Memory): 128 KB vs 8 GB

> 💡 데이터 저장하고 작업하는 공간의 크기

- 128 KB (ATmega128): 코드가 저장되는 공간(플래시 메모리)이 128KB입니다. 요즘 스마트폰으로 찍는 사진 한 장이 보통 3~5MB(3000~5000KB)인 것을 감안하면 매우 작은 용량입니다.
- 8 GB (Core i7): 8GB는 대략 8,388,608 KB입니다. ATmega128 메모리의 약 65,000배에 달합니다.
### 3. 클록 (Clock): 16 MHz vs 3.4 GHz (쿼드 코어)

> 💡 CPU의 일 처리 속도 (1초에 치는 박수 횟수)

- 16 MHz (ATmega128): 메가(M)는 백만을 뜻합니다. 즉, ATmega128은 1초에 1,600만 번의 신호(박수)를 만들며 일을 합니다.
- 3.4 GHz 쿼드 코어 (Core i7): 기가(G)는 10억을 뜻합니다. 3.4GHz는 1초에 34억 번 박수를 친다는 의미입니다. 게다가 이런 손(Core)이 4개(Quad)나 있어서 동시에 일을 나눠서 합니다.

> 💡 직렬 통신이란? 데이터를 1개의 선(Line)을 통해 한 비트씩 차례대로 보내는 통신 방식입니다. ATmega128A에서 외부 장치와 대화할 때 가장 많이 씁니다.

- 대표 예시: SPI 통신, I2C 통신

# 🧠 CPU 아키텍처 비교: CISC vs RISC

> 💡 핵심 요약

## 📊 한눈에 보는 핵심 비교 테이블

| 구분 | CISC (Complex Instruction Set Computer) | RISC (Reduced Instruction Set Computer) |
|---|---|---|
| 명령어 종류 | 매우 많고 복잡함 | 적고 단순함 |
| 명령어 길이 | 가변형 (명령어마다 길이가 다름) | 고정형 (모든 명령어 길이가 같음) |
| 설계 철학 | "소프트웨어가 짜기 편하게 하드웨어를 복잡하게 만들자" | "하드웨어를 가볍게 유지하고 소프트웨어로 해결하자" |
| 처리 속도 | 명령어 하나를 처리하는 데 여러 클록 소요 | 명령어 하나를 1클록에 처리 (파이프라이닝 최적화) |
| 전력 및 발열 | 높음 (회로가 복잡해서 전기를 많이 먹음) | 낮음 (회로가 단순해서 저전력/저발열에 유리) |
| 하드웨어 크기 | 칩의 구조가 크고 복잡함 | 칩의 구조가 작고 효율적임 |
| 대표 제품 | Intel Core 시리즈, AMD Ryzen (PC/서버용) | ATmega128A (AVR), Apple M 시리즈, ARM 계열 (스마트폰/임베디드) |

## 🔍 비유로 이해하는 CISC와 RISC

명령어 집합을 설계하는 철학의 차이를 이해하면 쉽습니다.

- 🏛️ CISC는 '다기능 만능 칼'입니다.
- 🔪 RISC는 '잘 드는 기본 식칼'입니다.

# 🏗️ 컴퓨터 구조의 양대산맥: 폰노이만 구조 vs 하버드 구조

> 💡 핵심 요약

## 📊 한눈에 보는 핵심 비교 테이블

| 구분 | 폰노이만 구조 (Von Neumann Architecture) | 하버드 구조 (Harvard Architecture) |
|---|---|---|
| 메모리 구성 | 명령어와 데이터가 하나의 메모리에 있음 | 명령어 메모리와 데이터 메모리가 물리적으로 분리됨 |
| 데이터 통로 (Bus) | 통로가 1개 (공유함) | 통로가 2개 (명령어 버스 / 데이터 버스 분리) |
| 작동 방식 | 명령어를 읽을 때는 데이터를 읽을 수 없음 (교대 처리) | 명령어를 읽는 동시에 데이터를 읽고 쓸 수 있음 (동시 처리) |
| 처리 속도 | 비교적 느림 (병목 현상 발생 가능) | 비교적 빠름 (고속 처리에 유리) |
| 하드웨어 복잡도 | 구조가 단순하고 비용이 저렴함 | 구조가 복잡하고 회로 설계 비용이 더 듦 |
| 대표 예시 | 데스크톱 PC (Intel, AMD 등), 범용 컴퓨터 | ATmega128A (AVR), DSP, 스마트폰 AP의 일부 구조 |

## 🔍 그림과 비유로 이해하기

### 좁은 1차선 도로 vs 뚫려있는 2차선 도로

- 🛣️ 폰노이만 구조는 '1차선 왕복 도로'입니다.
- 🛣️ 하버드 구조는 '명령어 전용 차선'과 '데이터 전용 차선'이 분리된 고속도로입니다.
## ⚙️ ATmega128A 스펙으로 직접 확인하는 하버드 구조

우리가 공부할 ATmega128A의 스펙을 보면 이 하버드 구조가 눈에 바로 보입니다. 앞서 정리했던 메모리 스펙을 다시 보면 이해가 쉽습니다.

- 플래시 메모리 (Flash Memory) - 128 KB: 내가 작성한 C언어 소스코드(명령어)가 저장되는 전용 공간입니다.
- SRAM - 4 KB: 코드 내부에서 쓰이는 변수나 데이터가 실시간으로 저장되고 삭제되는 전용 공간입니다.
> 💡 만약 ATmega128A가 폰노이만 구조였다면 그냥 '통합 메모리 132 KB' 구조였겠지만, 하버드 구조를 채택했기 때문에 코드 공간(128KB)과 데이터 공간(4KB)의 메모리가 물리적으로 완전히 쪼개져 있는 것입니다.

# 💾 메모리의 종류: ROM, RAM, SRAM, DRAM 완전 정복

> 💡 핵심 요약

## 📊 한눈에 보는 핵심 비교 테이블

| 구분 | ROM (Read Only Memory) | RAM (Random Access Memory) |
|---|---|---|
| 휘발성 여부 | 비휘발성 (전원이 꺼져도 유지) | 휘발성 (전원이 꺼지면 소멸) |
| 주요 역할 | MCU의 전원이 켜졌을 때 실행할 소스코드 저장 | 프로그램 실행 중 필요한 임시 변수/데이터 저장 |
| 속도 | 상대적으로 느림 | 상대적으로 매우 빠름 |
| ATmega128A 예시 | 플래시 메모리 (128 KB) | 내부 SRAM (4 KB) |

## 🔍 RAM의 두 형제: SRAM vs DRAM 상세 비교

RAM은 만드는 방식과 회로 구조에 따라 크게 SRAM과 DRAM으로 나뉩니다.

| 항목 | SRAM (Static RAM) | DRAM (Dynamic RAM) |
|---|---|---|
| 특징 | 정적 RAM (전원만 있으면 데이터 유지) | 동적 RAM (시간이 지나면 데이터가 사라짐) |
| 핵심 구조 | 플립플롭 회로 (트랜지스터 4~6개 사용) | 커패시터(축전기) 1개 + 트랜지스터 1개 사용 |
| 리프레시(재충전) | 필요 없음 | 주기적으로 필수 (방전되면 데이터 손실) |
| 속도 | 압도적으로 빠름 | SRAM에 비해 느림 |
| 용량 및 가격 | 구조가 복잡해 대용량이 어렵고 비쌈 | 구조가 단순해 대용량이 가능하고 저렴함 |
| 주요 사용처 | CPU 내부 캐시 메모리, MCU(ATmega128A) 내부 RAM | PC 메인 메모리 (우리가 흔히 사는 8GB, 16GB RAM) |

## ⚙️ 개념 상세 설명

### 1. ROM: 지워지지 않는 기억 장치

- 컴퓨터를 끄면 하드디스크의 파일이 그대로 남아있듯, MCU의 전원을 꺼도 우리가 업로드한 C언어 코드가 지워지면 안 됩니다. 이 코드가 저장되는 공간이 바로 ROM(그중에서도 플래시 메모리)입니다.
- 이름은 '읽기 전용'이지만, 컴퓨터에서 다운로더(ISP)를 통해 코드를 구울 때(Write)는 데이터를 쓸 수 있습니다. 단, 실행 중인 코드가 실시간으로 값을 바꾸지는 못합니다.
### 2. SRAM: 속도로 승부하는 메모리

- ATmega128A 내부에 들어있는 RAM입니다. 전기가 공급되는 동안에는 데이터가 완벽하게 유지됩니다.
- 회로가 복잡해서 칩의 공간을 많이 차지하기 때문에 ATmega128A에는 딱 4 KB라는 아주 작은 용량만 들어있습니다. 대신 속도가 CPU와 맞먹을 정도로 엄청나게 빨라 MCU 제어에 최적입니다.
### 3. DRAM: 가성비와 용량으로 승부하는 메모리

- 내부의 아주 작은 배터리(커패시터)에 전기를 채워서 데이터를 기억하는 방식입니다. 문제는 이 전기가 순식간에 자연 방전됩니다.
- 그래서 데이터가 안 날아가도록 1초에 수백 번씩 전기를 다시 채워주는 '리프레시(Refresh)' 작업을 하드웨어적으로 계속 해줘야 합니다. 제어는 복잡하지만 칩을 작고 싸게 만들 수 있어, PC처럼 기가바이트(GB) 단위의 대용량 메모리가 필요할 때 사용합니다.
