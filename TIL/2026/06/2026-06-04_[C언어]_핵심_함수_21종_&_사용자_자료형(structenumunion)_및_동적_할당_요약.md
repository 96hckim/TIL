### 📊 3대 키워드 핵심 요약

- auto (일반 변수의 기본값)
- register (CPU를 이용한 초고속 변수)
- static (메모리 붙박이 변수)
### 📌 변수 기억 부류 최종 비교표

| 구분 | 📊 auto | ⚡ register | 📌 static |
|---|---|---|---|
| 저장 장소 | 메모리 (RAM 스택) | CPU 내부 레지스터 | 메모리 (RAM 데이터) |
| 처리 속도 | 보통 | 지구상에서 가장 빠름 | 보통 |
| 주소 값(&) | 가능 (&num ⭕) | 절대 불가능 (❌) | 가능 (&count ⭕) |
| 수명 | 함수 종료 시 즉시 소멸 | 함수 종료 시 즉시 소멸 | 프로그램 종료 시까지 생존 |
| 기본 초기화 | 쓰레기 값 | 쓰레기 값 | 0으로 자동 초기화 |

### 📋 한 줄 요약

> auto는 금방 사라지는 일반 변수, register는 주소(&)를 딸 수 없는 CPU 전용 초고속 변수, static은 프로그램이 끝날 때까지 값을 기억하고 살아남는 붙박이 변수이다!



## 1. struct (구조체 기본형)

여러 종류의 서로 다른 자료형(int, char, double 등)들을 하나로 묶어 새로운 데이터 껍데기를 만드는 문법입니다.

### 💻 선언 및 사용 예시

```c
// 'Student'라는 이름의 설계도를 만듦
struct Student {
    int id;
    char name[20];
    double grade;
};

int main(void){
    // ⚠️ 변수를 만들 때 매번 앞에 'struct'를 꼭 붙여야 함
    struct Student s1 = { 202601, "홍길동", 4.5 };

    // 점(.) 멤버 연산자로 접근
    printf("학번: %d, 이름: %s\n", s1.id, s1.name);
    return 0;
}
```

- 현업 한 줄 정의: 관련 있는 개인 신상정보나 기기 상태 값들을 한 바구니에 담아 배달하는 상자와 같습니다. 다만, 변수를 만들 때마다 struct 단어를 타이핑해야 해서 단독으로는 귀찮음이 유발됩니다.
## 2. typedef struct (구조체 축약형)

기본 구조체의 치명적인 단점인 "변수 선언 시 struct 타이핑 필수" 규칙을 없애기 위해 자료형에 짧은 별명(Alias)을 붙여주는 실무 표준 문법입니다.

### 💻 선언 및 사용 예시

```c
// 구조체를 정의함과 동시에 맨 뒤에 짧은 단축 별명(Student_t)을 지정
typedef struct {
    int id;
    char name[20];
    double grade;
} Student_t; // 관례적으로 타입(Type)이라는 의미로 뒤에 _t를 많이 붙임

int main(void){
    // 😎 struct 단어를 떼고, 일반 int나 double처럼 깔끔하게 선언 가능!
    Student_t s1 = { 202602, "이순신", 4.3 };
    Student_t s2;

    return 0;
}
```

## 3. enum (열거형)

여러 개의 연속된 정수형 상수들을 사람이 읽기 쉬운 단어(문자열 형태)로 열거하여 이름을 붙여주는 문법입니다. 컴퓨터는 내부적으로 숫자로 읽고, 사람은 글자로 읽어 코드의 가독성을 극대화합니다.

### 💻 선언 및 사용 예시

- 현업 한 줄 정의: 구조체 선언 코드를 미니멀하게 줄여 자바나 C++의 클래스 변수를 선언하는 것처럼 트렌디하게 코딩할 수 있게 돕는 치트키입니다. 자기 참조 구조체(연결 리스트)를 짜는 게 아니라면 실무 소스코드는 99% 이 방식을 씁니다.
```c
// 아무 값도 안 주면 맨 앞 단어부터 0, 1, 2, 3... 숫자가 1씩 늘어나며 자동 배정됨
enum Week {
    SUNDAY,    // 0
    MONDAY,    // 1
    TUESDAY,   // 2
    WEDNESDAY  // 3
};

// 원한다면 수동으로 숫자를 지정할 수도 있음 (보통 가독성을 위해 대문자 명명)
enum Menu {
    MAKE_ACC = 1,
    DEPOSIT = 2,
    WITHDRAW = 3,
    EXIT = 9
};
```

- 현업 한 줄 정의: 코드 내부에서 if (status == 3)이라고 짜면 3이 무슨 의미인지 아무도 모릅니다. 대신 if (status == WITHDRAW)라고 짜면 누구나 출금 상태임을 바로 알 수 있죠. 즉, 의미 불명의 숫자(매직 넘버)들을 직관적인 단어로 치환하는 청정 가독성 도구입니다. (switch-case 문과 궁합이 환상적입니다.)
## 📊 3대 문법 핵심 요약 비교표 (노션 박제용)

| 구분 | 🧱 struct | 💎 typedef struct | 🏷️ enum |
|---|---|---|---|
| 핵심 목적 | 다양한 타입의 변수 묶기 | 구조체 변수 선언 코드 간소화 | 의미 있는 정수형 상수들의 집합 만들기 |
| 선언 예시 | struct 태그명 { ... }; | typedef struct { ... } 별명; | enum 이름 { 단어1, 단어2 }; |
| 변수 선언문 | struct 태그명 변수명; | 별명 변수명; | enum 이름 변수명; |
| 실무 활용 | 연결 리스트 노드, 상호 참조 데이터 구조 | 일반적인 모든 커스텀 데이터 모델, API 설계 | 프로그램 메뉴 번호, 하드웨어 상태 값 (State) |

### 📋 한 줄 요약

> • **struct**는 다른 종류의 데이터를 하나로 포장하는 상자이고,
• **typedef struct**는 그 상자를 다루기 편하게 '짧은 별명 이름표'를 달아주는 실무 표준이며,
• **enum**은 코드 속 정수 숫자들에게 인간의 언어로 된 '이름표'를 붙여 가독성을 올리는 상수 집합 장치이다!



## 1. 구조체(struct) vs 공용체(union) 메모리 차이

가장 핵심이 되는 메모리 구조의 차이입니다. 똑같은 멤버를 가져도 메모리에 배치되는 형태가 완전히 다릅니다.

### 🧱 구조체 (struct)

- 특징: 모든 멤버 변수가 자신만의 메모리 공간을 각자 따로 차지합니다.
- 크기: 모든 멤버들의 크기를 합친 만큼 커집니다. (패딩 바이트 제외 시)
### 👥 공용체 (union)

- 특징: 모든 멤버 변수가 가장 첫 번째 주소(시작 주소)를 공유하며 같은 메모리 공간을 번갈아 가며 씁니다.
- 크기: 멤버 중 가장 덩치가 큰(자료형 크기가 가장 큰) 멤버의 크기로 딱 고정됩니다.
## 2. 💻 문법 및 소스코드 예시

```c
#include <stdio.h>

// 공용체 정의 (struct 자리에 union만 쓰면 끝)
typedef union {
    char a;      // 1바이트
    int b;       // 4바이트
    double c;    // 8바이트 👈 가장 큰 멤버!
} MyUnion_t;

int main(void){
    MyUnion_t u;

    // 1. 공용체 전체 크기 확인
    // 멤버들의 합(1+4+8=13)이 아니라, 가장 큰 double 크기인 '8바이트'가 나옵니다.
    printf("공용체 u의 크기: %d 바이트\n\n", (int)sizeof(u));

    // 2. 모든 멤버의 시작 주소가 완벽히 똑같음 확인
    printf("a의 주소: %p\n", (void*)&u.a);
    printf("b의 주소: %p\n", (void*)&u.b);
    printf("c의 주소: %p\n\n", (void*)&u.c);

    // 3. ⚠️ 공용체 사용 시 주의점 (덮어쓰기 현상)
    u.b = 1234;
    u.c = 5.67; // 👈 가장 최근에 대입한 값만 살아남음!

    // u.b 자리에 u.c가 덮어씌워졌기 때문에 b를 출력하면 깨진 쓰레기 값이 나옵니다.
    printf("u.b의 값: %d (값이 오염됨)\n", u.b);
    printf("u.c의 값: %.2f (정상 출력)\n", u.c);

    return 0;
}
```

## 3. 🛠️ 이 위험한 걸 도대체 실무에선 언제 쓰나요?

"한 번에 하나의 멤버만 써야 하고 값이 오염된다면 왜 쓰는 거지?"라는 의문이 당연히 듭니다. 하지만 공용체는 임베디드 하드웨어 제어 및 네트워크 통신에서 최고의 사기 치트키로 쓰입니다.

### 💡 실무 치트키: "하나의 데이터를 다르게 해석하기" (비트 쪼개기)

하드웨어 센서나 네트워크에서 4바이트짜리 정수 데이터(int) 하나가 통째로 날아왔다고 가정해 봅시다. 그런데 이 데이터를 1바이트씩 쪼개서 [상위 1바이트, 중위 1바이트 ...] 형태로 각각 가공해야 할 때, 공용체를 쓰면 복잡한 비트 연산(<<, >>, &) 없이 주소 공유만으로 날로 먹을 수 있습니다.

```c
#include <stdio.h>

typedef union {
    int total_data;         // 4바이트 통짜 정수 데이터로 읽기
    char bytes[4];          // 1바이트짜리 4개 배열로 쪼개서 읽기
} Packet_t;

int main(void){
    Packet_t p;

    // 통신으로 4바이트 데이터를 한 번에 수신
    p.total_data = 0x12345678;

    // 공용체 공간이 공유되므로, 비트 연산 없이 각 바이트가 자동 슬라이싱되어 있습니다.
    printf("1번째 바이트: 0x%x\n", p.bytes[0]); // 0x78
    printf("2번째 바이트: 0x%x\n", p.bytes[1]); // 0x56
    printf("3번째 바이트: 0x%x\n", p.bytes[2]); // 0x42
    printf("4번째 바이트: 0x%x\n", p.bytes[3]); // 0x12 (리틀 엔디안 기준)

    return 0;
}
```

## 📊 struct vs union 최종 핵심 요약표

| 구분 | 🧱 구조체 (struct) | 👥 공용체 (union) |
|---|---|---|
| 메모리 배치 | 모든 멤버가 독립된 방을 씀 | 모든 멤버가 같은 방(주소)을 공유함 |
| 전체 크기 | 모든 멤버 크기의 합 (패딩 포함) | 가장 큰 멤버 자료형의 크기 |
| 값의 보존 | 모든 멤버의 값을 동시에 안전하게 보존 | 가장 최근에 저장한 단 하나의 멤버만 유효 |
| 주요 목적 | 다양한 데이터를 한 덩어리로 묶을 때 | 메모리 절약, 하나의 데이터를 다양한 형태(비트/바이트)로 해석할 때 |

### 📋 한 줄 요약

> **공용체(union)**는 모든 멤버가 동일한 메모리 시작 주소를 공유하는 자료형으로, 크기는 가장 큰 멤버를 따라가며, 임베디드 실무에서는 4바이트나 2바이트 데이터를 비트 연산 없이 각 바이트 단위로 쪼개어 분석할 때 강력한 위력을 발휘한다!



## 1. 🏭 동적 할당의 핵심 메모리: 힙(Heap) 영역

컴퓨터의 램(RAM) 메모리는 구역이 나뉘어 있습니다.

- 스택 (Stack) 영역: 일반 지역 변수가 살다 가는 곳. 함수가 끝나면 칼같이 파괴됨. (메모리 크기가 작음)
- 힙 (Heap) 영역: malloc이 메모리를 빌려오는 거대한 자유 공간. 개발자가 직접 명령하기 전까지는 절대로 값이 사라지지 않음.
## 2. 💻 malloc의 기본 문법 공식

malloc 함수를 사용하려면 #include <stdlib.h> 헤더 파일이 반드시 필요합니다.

```c
#include <stdlib.h>

// 💡 공식: 자료형* 포인터변수 = (자료형*)malloc(sizeof(자료형) * 개수);
int* arr = (int*)malloc(sizeof(int) * 5); // int형(4바이트)짜리 방 5개를 힙에 개설! (총 20바이트)
```

### 🔍 쪼개서 분석하기 (왜 이렇게 쓸까?)

1. sizeof(int) * 5 : 컴퓨터에게 "나 20바이트만큼 방 필요해"라고 정확한 바이트 크기를 계산해서 던져주는 것입니다.
1. (int*) (형변환/Casting) : malloc은 처음에 힙 영역에서 빈 방을 찾아 주소를 줄 때, 이 방이 int형 방인지 char형 방인지 모른 채 그냥 순수 주소(void*)만 툭 던져줍니다. 그래서 앞날에 이 주소를 int 자물쇠로 열고 닫을 수 있도록 (int*)라고 명확하게 강제 형변환(Casting)을 해주는 것입니다.
## 3. 🛠️ malloc 사용 4단계 표준 프로세스 (실무 양식)

동적 할당은 메모리를 직접 제어하는 만큼, 현업에서는 아래 4단계의 방어적 코드 규격을 무조건 준수합니다.

```c
#include <stdio.h>
#include <stdlib.h> // malloc, free를 쓰기 위한 필수 헤더

int main(void){
    // [1단계] 포인터 선언 및 메모리 할당
    int* p_arr = (int*)malloc(sizeof(int) * 3);

    // [2단계] 예외 처리 (★가장 중요★)
    // 컴퓨터 메모리가 꽉 찼거나 오류가 나면 malloc은 NULL(0) 주소를 반환합니다.
    if (p_arr == NULL) {
        printf("메모리 할당 실패!\n");
        return -1; // 프로그램 비정상 종료 처리
    }

    // [3단계] 일반 배열처럼 마음껏 사용하기
    p_arr[0] = 10;
    p_arr[1] = 20;
    p_arr[2] = 30;

    for (int i = 0; i < 3; i++) {
        printf("%d ", p_arr[i]);
    }
    printf("\n");

    // [4단계] 다 썼으면 반드시 메모리 반납 (★필수★)
    // 빌려온 메모리를 반납하지 않으면 메모리가 계속 좀먹는 '메모리 누수(Memory Leak)'가 발생합니다.
    free(p_arr);
    p_arr = NULL; // 댕글링 포인터 방지를 위해 주소를 완전히 초기화 (안전장치)

    return 0;
}
```

## 4. 💡 왜 굳이 귀찮게 malloc을 써야 할까? (실무 활용 이유)

### ① 함수가 끝나도 살아있는 주소를 리턴하기 위해

앞서 공부했던 내용이죠! 일반 지역 변수의 주소는 함수가 끝나면 파괴되어 반환할 수 없지만, malloc으로 만든 힙 영역의 방은 free()를 하기 전까지 영원히 살기 때문에 안전하게 함수 외부로 주소를 돌려줄 수 있습니다.

### ② 사용자 입력에 따라 배열 크기를 실시간으로 결정하기 위해

일반 배열인 int arr[n]; 구조는 컴파일할 때 크기가 고정되어야 해서 사용자 입력값 n을 배열 크기로 넣으면 컴파일 에러가 나거나 불안정합니다. malloc은 이게 가능합니다.

```c
int count;
printf("학생 수는 몇 명인가요? : ");
scanf("%d", &count);

// 사용자가 키보드로 입력한 숫자만큼 딱 맞춰서 실시간 배열 생성! (메모리 절약)
Student_t* std_list = (Student_t*)malloc(sizeof(Student_t) * count);
```

## 📊 일반 배열 (정적) vs malloc (동적) 최종 비교표

| 구분 | 🧱 일반 배열 (정적 할당) | 💎 malloc (동적 할당) |
|---|---|---|
| 할당되는 메모리 영역 | 스택 (Stack) 영역 | 힙 (Heap) 영역 |
| 크기 결정 타이밍 | 컴파일 시점 (코드 짤 때 이미 고정) | 런타임 시점 (프로그램 실행 중 결정) |
| 생명 주기 (수명) | 함수가 종료되면 자동으로 파괴됨 | 개발자가 free() 명령을 내릴 때까지 생존 |
| 메모리 해제 | 자동으로 처리됨 (신경 쓸 필요 없음) | 수동 처리 (free() 필수, 안 하면 누수 버그) |
| 공간의 한계 | 스택 영역이 작아 대용량 데이터는 튕김 | 램(RAM) 시스템 성능이 허용하는 한 거대하게 가능 |

### 📋 한 줄 요약

> **malloc**은 프로그램 실행 도중 원하는 바이트 크기만큼 힙(Heap) 공간에 실시간 메모리를 개설하는 치트키이며, 함수가 끝나도 파괴되지 않는 절대적 수명을 얻는 대신 반드시 사용 후 **free()**로 수동 해제를 해주어야 메모리 누수 버그를 막을 수 있다!
